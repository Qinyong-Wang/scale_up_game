# 端到端调试

> **目的**:在没有交互式 Godot 编辑器(CI、远程 agent、headless 排查)的前提下,验证一条 UI / 信号链 / 资产产出是否真的跑通——并且能拿到截图、字体覆盖、Label 实际字体这类"肉眼才能看出来"的状态。
> **读者画像**:正在 debug 一个"逻辑测试都过但跑起来不对"的 bug 的人;在 agent 里没有 GUI 控制权的 Claude Code。
> **状态**:🟡 草稿。第一版基于"字体 tofu"那次 debug 沉淀。

---

## 1. 三类问题、三种工具

|问题类型|症状|首选工具|
|---|---|---|
|**资源本身**对不对(字体有没有字形、`.tres` 字段对不对)|"应该有的东西没有"|`tools/font_diag.gd` 这类一次性 SceneTree 脚本|
|**运行时绑定**对不对(Label 实际拿到哪个 Font、节点 theme 是不是预期)|"加载了但生效的不是这个"|`AGI_DIAG=1` 启动模式|
|**最终渲染**对不对(布局、颜色、字形)|"截图才看得出来"|`AGI_SCREENSHOT=1` + Read PNG|

按这个顺序排,大多数 visual bug 都能定位到具体一层。

## 2. `AGI_*` 环境变量启动模式

`scenes/main/main.gd` 的 `_ready()` 末尾根据三个环境变量切换分支,**正常游玩时不存在副作用**。

```bash
# 把 Label 实际拿到的 Font 与 CJK 覆盖打到 stdout 后退出
AGI_DIAG=1 godot --path .

# 自动开一次预训练 + 推 3 回合(配合 SCREENSHOT 用,看任务完成后的状态)
AGI_AUTOPLAY=1 godot --path .

# 启动后等 8 帧抓 viewport 截图存 user://screenshot.png 退出
AGI_SCREENSHOT=1 godot --path .

# 组合:跑完一次预训练再截图,看模型库 / 任务面板的最终视觉
AGI_AUTOPLAY=1 AGI_SCREENSHOT=1 godot --path .

# 截图前先切到指定侧栏 tab (验收 §10 step 4+ 的新 sidebar dashboard UI)
AGI_AUTOPLAY=1 AGI_SCREENSHOT=1 AGI_INITIAL_NAV=models godot --path .

# 截「起始页(主菜单)」本身 — 上面的 AGI_SCREENSHOT 那条路径会跳过菜单直接进游戏,
# 截不到主菜单。这个开关由 scenes/start_screen/start_screen.gd 处理: 正常建菜单 →
# 渲染几帧落定 → 存 user://start_screen_shot.png 退出。验收 hero 化主菜单视觉用。
AGI_SHOT_MENU=1 godot --path .
```

侧栏 nav_id 列表 (`main.gd::SIDEBAR_NAV`):

| 组 | nav_id |
|---|---|
| 运营 | `overview` / `economy` / `hiring` / `tasks` / `events` |
| 研发 | `models` / `infra` / `dataset` / `tech` |
| 市场 | `product` / `marketing` / `market_rank` / `monetization` |

### 2.1 截图 wrapper: `tools/screenshot.sh`

一行命令出图, 自动清旧 + 跑 autoplay + 切 tab + 输出截图绝对路径:

```bash
tools/screenshot.sh                          # 默认 overview
tools/screenshot.sh models                   # 模型 tab
tools/screenshot.sh infra                    # 基建 tab
tools/screenshot.sh hiring|product|dataset|tasks|events|tech|...
tools/screenshot.sh drawer:deploy_first_dc   # 抽屉验收: 切 infra + 自动打开部署抽屉
```

`drawer:<scenario>` 把抽屉撑开后再截图, 验收 `Drawer` 组件的实际渲染。当前支持的 scenario:

| scenario | 触发 |
|---|---|
| `deploy_first_dc` | 切 `infra` tab, 找第一个 idle DC, 调 `_open_deploy_drawer_for_dc` 打开"部署模型"抽屉, 列出玩家已发布模型 + 公共开源模板 |

新场景在 `main.gd::_open_drawer_for_screenshot()` match 块里加分支。

Claude Code agent 用这条命令不会触发 `rm` 的 ask 询问 (script 内部统一处理)。

截图位置(macOS):

```
~/Library/Application Support/Godot/app_userdata/Scaling Up/screenshot.png
```

Claude Code 直接 `Read` 这个路径就能看图。其他平台路径可由日志最后一行的 `path=...` 拿到 (script 也把绝对路径输出到 stdout)。

> **不能用 `--headless` 截图**:headless 没有 GL/Metal context, viewport 渲染不出字形,只会得到一张空白底色图。截图必须让 Godot 起真窗口。

> **why these flags live in `main.gd`**:`tools/screenshot.gd` 那种走 `-s tools/foo.gd` 的 `SceneTree` 旁路启动方式**不会加载 autoload**, EventBus / GameState 全部不存在,业务场景一加载就炸。env flag 的好处是走的是项目正常启动路径,跟玩家点 F5 完全一致,只是 `_ready` 里多走一个分支。

### 2.2 卡片墙视觉验收清单

每个迁过的 tab 在 autoplay 后的预期视觉特征 (见 `design/UI视觉系统设计.md` §11 验收标准):

| Tab | 预期看到 |
|---|---|
| `overview` | 顶栏 5 chip (回合/现金/周净流/付费用户/算力) + 侧栏 4 组 18 项 + 概览看板 |
| `models` | SectionHeader "我的模型 N" + 5 pill FilterBar + 搜索化名 + 排序 + ≥1 Card (Avatar 哈希配色 + 状态 Badge + 4 action) |
| `hiring` | 创始人区 + 候选池/已签约按 specialty 分组的 LeadCard 网格 + 员工 +1/-1 行 + 周工资合计 |
| `infra` | DC HFlow (Avatar 显示首字 + 状态 Badge + 容量/TFLOPs/t/s/周成本/在跑) + 自建队列 (有时无) |
| `product` | 创建按钮 / 算力池 util 彩条 (capacity=0 红 / >80% 黄 / >100% 红) + ProductCard (API vs Chatbot 字段集不同) |
| `dataset` | 预训练/后训练 kind 切换 + 市场列表 + 自采按钮 + 我的数据集 |
| `tasks` | 进行中任务 cards 含 ProgressBar (autoplay 后通常为空) |
| `events` | 待处理事件 cards + debug 触发 + 历史 |
| `tech` | 5 棵子树 (arch / attention / loss / engineering / application), 已解锁 ✓ 行 + 可研究/研究中 cards |
| `drawer:deploy_first_dc` | 主区被压缩到左侧, 右侧 360px Drawer 显示标题 "部署模型 → <dc_id>" + 模型选择列表 (我的已发布 + 公共开源) + 左下"取消"按钮 |

**常见视觉 bug 与排查**:

- 卡片极扁 / 文字重叠: 组件 `extends Control` + 内层 `PRESET_FULL_RECT` 不能向父 Container 冒泡最小尺寸。改 `extends HBoxContainer/PanelContainer` 让 `get_combined_minimum_size` 自动算。
- 卡片被横向拉成宽条: `size_flags_horizontal = SIZE_FILL` 在 HFlow 里会全宽。改 `SIZE_SHRINK_BEGIN`。
- 卡片整列高度 0 / 不可见: 内层嵌了多余 ScrollContainer 被外层 `_make_tab` 的 ScrollContainer 压扁。view 内部不要套二层 scroll。
- 组件文案为空 (set_data 失效): 在 `_ready` 跑之前调 set_*, 内部 Label 还没建。组件需要存 `_pending_*` 等 `_ready` 时再 apply。

## 3. 一次性诊断脚本(`tools/*.gd`)

适合**纯数据查询**——不需要场景树、不需要 autoload。模板:

```gdscript
extends SceneTree

func _init() -> void:
    var res := load("res://path/to/asset")
    # ...各种检查...
    print("...")
    quit(0)
```

跑法:

```bash
godot --headless --path . -s tools/<name>.gd
```

现存:

- `tools/font_diag.gd` — 加载配置的 UI 字体, 列指定码点的 `has_char` 结果, 验证 `UITheme` 安装后的默认控件字体是否生效。

新加诊断脚本时**不要**在里面引用 EventBus / GameState / 业务 system,那些是 autoload,SceneTree 旁路启动里都不在。要查跨系统行为请走单测或 `AGI_DIAG`。

## 4. 通用流程:visual / 信号链 bug 怎么定位

1. **先跑单测**确认逻辑层没问题。如果单测红,根本不是 e2e 问题,回 TDD。
2. **`AGI_AUTOPLAY=1 AGI_SCREENSHOT=1` 跑一次**,Read 截图,肉眼比对预期。
3. **如果数值对、视觉错**(布局错位、字符 tofu、颜色不对):
   - 先查"运行时绑定":写个 `_diag` 分支或加临时 `print`,把 Label 的 `get_theme_font(&"font").get_font_name()` 这类**实际拿到的资源**打出来。
   - 再查"资源本身":写一个 `tools/<thing>_diag.gd` 验证资产的属性,排除"资产本身就不对"。
4. **如果数值不对**(任务进度条没动、模型库不刷新):看日志(`Log.captured` 在测试态保留;运行态走 console)。`AGI_AUTOPLAY` 自带几条 `[INFO]` 输出,用来卡进度。
5. **修完务必删掉**临时 `print` / `_diag` 中的脏代码;但 `AGI_*` 这套基础脚手架**保留**,下一次 bug 就能直接复用。

## 5. 已知字体陷阱

`gui/theme/default_font="res://...ttf"` 在 Godot 4.3 macOS 上**不会稳定生效**, Label 仍可能拿 `Open Sans SemiBold`(内置)。CJK 立刻 tofu。

正确做法分两层:

- `scripts/autoload/ui_theme.gd` 在 autoload 阶段把 CJK 字体装进 `ThemeDB.get_default_theme()`,保证新建 Label/Button 在设置中文 text 前已经拿到正确字体。
- `scenes/main/main.gd` 的 `_apply_theme()` 仍给 root Control 设显式 `Theme`,作为主 HUD 的视觉权威。
- GUT runner / 编辑器面板必须走 `tests/support/gut_cjk_font_hook.gd`; 这个 hook 同时覆盖 `RichTextLabel` 和 `TextEdit`, 否则编辑器底部 GUT 输出中文失败信息时仍会用插件自带 Courier 字体触发空系统 fallback。

不要在 UI 文案里使用 `⚠` 这类当前占位字体缺失的符号;用 `注意:` 这类文本前缀,避免 Godot 触发空字体 fallback 并刷 `FreeType: Error loading font: ''`。

字体来源现状: `assets/fonts/cjk.ttf` 是 macOS 自带 `Arial Unicode.ttf` 的拷贝, **私有许可不可分发**,只是开发期占位。发版前换 Noto Sans CJK SC,详见 `assets/fonts/README.md`。

### 5.1 编辑器在 macOS 26 上卡死（系统字体 fallback 返回空字体）—— 已随升级 4.4.1 解决

> **状态: ✅ 已解决。** 本工程已从 Godot 4.3 升级到 **4.4.1**，4.4 修复了 macOS 26 的系统字体 fallback bug；编辑器、`--editor --quit`、`--import` 均恢复正常。本节保留作为"为什么必须用 4.4+"的根因记录——**若被迫退回 4.3 + macOS 26，下述卡死会复现**。

**症状（4.3 + macOS 26）**: `godot --headless --editor --quit`(以及 `--headless --import` 和图形编辑器)**永久卡死**, 一核 100% CPU, 刷数万条:

```
ERROR: FreeType: Error loading font: ''.   at: _ensure_cache_for_size
ERROR: Condition "!_ensure_cache_for_size(...)" is true.   at: _font_has_char
Failed method: EditorToaster::_error_handler_impl. Message queue out of memory.
Failed method: EditorNode::_print_handler_impl. Message queue out of memory.
```

**根因(已坐实)**: 这是 **Godot 4.3 × macOS 26 的引擎/系统不兼容**, 不是工程 bug。macOS 26 改了 CoreText 系统字体 API, Godot 4.3 (2024 年中) 在文字 shaping 需要"已加载字体里没有的字形"(比如用拉丁字体渲染中文)时, 走系统字体 fallback 拿到的是**空字体** → 每个缺字报一次 `FreeType: Error loading font: ''`。在编辑器里这被 `EditorToaster` 放大: 报错 → toaster 重新渲染报错文本 → 又触发 shaping → 又报错 → 消息队列被塞爆 → 卡在单次 `Main::iteration()` 出不来, 永远到不了 `--quit`。

最小复现(无编辑器、无 toaster, 只证明 fallback 返回空字体):

```gdscript
extends SceneTree
func _init() -> void:
	var f := ThemeDB.get_fallback_font()   # 内置仅拉丁字体
	var rid := TextServerManager.get_primary_interface().create_shaped_text()
	TextServerManager.get_primary_interface().shaped_text_add_string(rid, "中文测试", f.get_rids(), 16)
	TextServerManager.get_primary_interface().shaped_text_shape(rid)   # → 刷 FreeType: '' 
	quit(0)
```

**为什么跑测试不受影响**: `gut_cmdln` 走 SceneTree、没有 `EditorToaster` 放大器, 且 `tests/support/gut_cjk_font_hook.gd` 预先装了 `cjk.ttf` 覆盖所有中文字形 → 0 错误。卡死**只发生在编辑器上下文**(`--editor` / `--import`)。

**排查时排除过但都不是触发点**: 打开的脚本标签(`script_editor_cache.cfg`)、外部编辑器配置(`text_editor/external`)、GUT 编辑器插件、把编辑器 `main_font`/`code_font` 改成 cjk.ttf —— 逐一关掉后循环照旧。字体导入产物(`.godot/imported`)全都健康, 删 `.godot` 重建也没用。只要编辑器渲染**任何**中文就会撞引擎 bug, 而本工程到处是中文。

**最终解法（已采用）**: 升级到 **Godot 4.4.1**。在 4.4 上跑上面的最小复现 → `glyph_count` 正常、0 条 `FreeType: ''`，系统 fallback 已修。
- 安装与软链见 `docs/开发环境配置.md §2.1`（4.4.1 装在 `/Applications/Godot_4.4.1.app`，旧 4.3 保留可回滚）。
- 升级时顺带修了一处 4.4 更严格的 CSV 解析报错：`resources/i18n/strings.csv` 里 6 行译文含未加引号的逗号（4.3 容忍、4.4 报错），已补引号；规范见 [[content-i18n-source-string-keys]] 思路（含逗号的字段必须用 `"…"` 包裹）。
- 重建 class 缓存恢复用标准命令 `godot --headless --path . --editor --quit`（4.4 上几秒干净退出）。
