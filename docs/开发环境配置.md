# 开发环境配置

> **目的**：把一台干净的 macOS 开发机搭到能打开工程、跑测试、调试的状态。
> **读者画像**：第一次拉这个仓库的人 / 换机后重装的人 / Claude Code。
> **状态**：🟡 草稿。

---

## 1. 工具清单

| 工具 | 版本 | 用途 | 必装 |
|---|---|---|---|
| Godot Engine | **4.4.1 stable**（GL Compatibility） | 引擎 + 编辑器 + CLI | ✅ |
| Godot 导出模板 | 与引擎同版本 | 打包用，详见 `构建与发布.md` | 🔶 仅打包时 |
| Git | ≥ 2.30 | 版本管理 | ✅ |
| VS Code + godot-tools | godot-tools **≥ v2.5**（4.4 LSP 已补 `range` 字段） | 外部编辑 .gd / 断点调试 | 🔶 推荐 |
| GUT 插件 | 9.x（Godot 4 兼容版） | 单元 / 集成测试 | ✅ |

> **为什么用 4.4.1（macOS 26 必需）**：Godot 4.3 的编辑器在 macOS 26 上一启动就因系统字体 fallback bug 死循环卡死（`docs/端到端调试.md §5.1`），无法开编辑器/重建缓存。4.4.1 修复了该 bug。`project.godot` 的 `config/features` 标记为 `"4.4"`。要再换大版本请先在 `design/` 写迁移说明再走 TDD 流程。

## 2. 安装步骤（macOS）

### 2.1 Godot 4.4.1

任选其一：

```bash
# 方式 A：Homebrew
brew install --cask godot

# 方式 B：官网/归档下载（确保拿的是 standard 版，不是 .NET 版）
# https://godotengine.org/download/archive/4.4.1-stable/
# 或 GitHub: godotengine/godot-builds → Godot_v4.4.1-stable_macos.universal.zip
# 解压后把 Godot.app 拖进 /Applications；下载来的需清隔离: xattr -dr com.apple.quarantine /Applications/Godot.app
```

为了 CLI 调用方便，建议把 Godot 二进制软链到 `PATH`：

```bash
ln -sf /Applications/Godot.app/Contents/MacOS/Godot /usr/local/bin/godot
godot --version   # 期望输出：4.4.1.stable.official
```

如果你把 Godot 放在其他路径，只要保证 `godot --version` 能在仓库目录里输出 `4.4.1.stable.official` 即可。

### 2.2 拉取并导入工程

```bash
git clone <repo-url> scaling-up
cd scaling-up
godot --headless --import   # 首次构建 .godot/ 缓存，跑完会退出
```

之后用 GUI 打开工程：`Godot → Scan → 选择本目录 → Import & Edit`。

### 2.3 GUT 测试插件

GUT 不能进 git（按 `.gitignore` 我们也不想把 `addons/` 全交上去），但需要每个开发者本地有：

```bash
# 推荐：装在工程内 addons/ 下，由 Godot 编辑器启用
mkdir -p addons
cd addons
git clone --depth 1 --branch godot_4 https://github.com/bitwes/Gut.git gut

# 启用插件：Godot → Project → Project Settings → Plugins → GUT → Enable
```

> **为什么用 git clone 而不是 AssetLib**：CI 里要无 GUI 装回来，clone 更稳。
> **为什么放 addons/gut/ 这个固定路径**：GUT 默认查这个路径，改了要同步改 `.gutconfig.json`。

`addons/gut/` 已在 `.gitignore`，不要把插件源码提交进仓库。干净 clone 第一次导入工程时，如果还没安装 GUT，Godot 可能提示找不到 `res://addons/gut/plugin.cfg`；安装插件后重新打开工程即可。

### 2.4 VS Code（可选但推荐）

1. 装扩展（4.4 起用最新版即可；当年锁 v2.4.0 是因为 v2.5+ 在 **Godot 4.3** LSP 上会抛 `Cannot read properties of undefined (reading 'range')`——4.3 的 LSP 返回的 symbol 不全带 `range` 字段，4.4+ 已补上）：

   ```bash
   code --install-extension geequlim.godot-tools
   ```

2. 在 Godot 里：`Editor → Editor Settings → Text Editor → External` 勾上 `Use External Editor`，路径设为 `command -v code` 输出的绝对路径（不要留空；例如 `/Users/<you>/.local/bin/code`），参数设为 `{project} --goto {file}:{line}:{col}`
3. 在 VS Code 里：`F1 → Godot Tools: Open workspace with Godot editor` 一次，建立 LSP 连接

断点调试：在 `.gd` 里下断点，按 `F5`，会自动 attach 到正在运行的 Godot 实例。

## 3. 跑测试

### 3.1 命令行（开发循环 / CI 都用这个）

```bash
# 开发循环：先跑单元测试，当前应无失败、无 pending、无 orphan
godot --headless --path . -s addons/gut/gut_cmdln.gd -gdir=res://tests/unit -gexit

# 发布前：再跑集成测试，退出码必须是 0
godot --headless --path . -s addons/gut/gut_cmdln.gd -gdir=res://tests/integration -gexit

# 需要一次性扫全套时也可以跑根目录
godot --headless --path . -s addons/gut/gut_cmdln.gd -gdir=res://tests -gexit

# 只跑某个文件（用 -gselect 按文件名过滤；别用 -gtest，.gutconfig 的 dirs 仍会拉全套）
godot --headless --path . -s addons/gut/gut_cmdln.gd -gselect=economy_test.gd -gexit

# 详细输出（看每个 assert）
godot --headless --path . -s addons/gut/gut_cmdln.gd -gdir=res://tests/unit -glog=3 -gexit
```

退出码：通过 `0`，失败 `1`。CI 里直接把退出码当结果。

> 当前集成套件仍可能打印少量已知 pending/risky/orphan 诊断；只要退出码为 `0`，表示现阶段集成回归通过。发布前仍应记录这些遗留项，避免把新 warning 混进去。

> 建议把上面这串包成一个 shell 别名或 Makefile 目标，目前没必要写，先记在这里。

### 3.2 编辑器内（调试单个用例）

`Bottom Panel → GUT → Run All` / `Run Current`。失败时点报错跳源码。

### 3.3 测试目录约定

```
tests/
  unit/                 单系统、单脚本
    <system>_test.gd
    ui_views/           各 tab view 的 UI 测试（见下方注）
  integration/          跨系统 + 回合推进 + EventBus 端到端
    <scenario>_test.gd
  support/              GUT pre-run hook（gut_cjk_font_hook.gd 等）
```

文件名必须以 **`_test.gd` 结尾**（`.gutconfig.json` 的 `suffix`），GUT 才会扫到。view 的 UI 测试放 `tests/unit/ui_views/`，且别用 `name` 当测试函数参数名（守护测试会挂）。

### 3.4 新增 `class_name` 后重建脚本类缓存

`gut_cmdln.gd` 走 SceneTree、不会刷新脚本类缓存。**新增一个带 `class_name` 的脚本**后，`gut_cmdln` 可能刷 `Could not find type ...` 一类级联解析错——因为 `.godot/global_script_class_cache.cfg` 还没登记这个新类。重建一次再跑测试：

```bash
godot --headless --path . --editor --quit   # 重建类缓存，几秒退出
godot --headless --path . -s addons/gut/gut_cmdln.gd -gdir=res://tests -gexit
```

测试出现一片莫名失败时，先怀疑类缓存过期、重建一次，而不是逐个改测试。

> **历史坑（已随 4.4 升级解决）**：在 **Godot 4.3 + macOS 26** 上，`--editor --quit` / `--import` 会因系统字体 fallback bug 永久卡死（详见 `docs/端到端调试.md §5.1`）。这正是本工程从 4.3 升到 4.4.1 的原因。若你被迫退回 4.3，这条命令会挂——别在那种组合下用。

## 4. 工作流速查

按 `CLAUDE.md` 的 TDD 流程，每改一个功能：

1. 改 `design/<对应系统>.md`
2. 写 / 改 `tests/...` 让它先红
3. 改 `scripts/` `scenes/` `resources/` 让它绿
4. 跑单测：`godot --headless ... -gdir=res://tests/unit -gexit`
5. 跑集成测：`godot --headless ... -gdir=res://tests/integration -gexit`
6. `git commit`

## 5. 常见问题

| 现象 | 原因 | 处理 |
|---|---|---|
| `godot: command not found` | `/usr/local/bin/godot` 软链没建 | 重做 §2.1 末尾那条 `ln -sf` |
| 打开工程时一堆 `.import` 报错 | 第一次没跑 `--headless --import` | `cd 工程目录 && godot --headless --import` |
| GUT 面板不出现 | 插件没启用 | Project Settings → Plugins → GUT → Enable |
| GUT 面板显示中文结果时刷 `FreeType: Error loading font: ''` | 插件输出控件还在用 Courier/Anonymous 字体 | 确认 `.gutconfig.json` 有 `pre_run_script=res://tests/support/gut_cjk_font_hook.gd`, 重启 Godot 后重跑 |
| 编辑器 / `--editor --quit` / `--import` 卡死 100% CPU，狂刷 `FreeType: Error loading font: ''` + `EditorToaster ... Message queue out of memory` | 你在用 **Godot 4.3 + macOS 26**：系统字体 fallback 返回空字体，渲染中文时被 toaster 放大成死循环（`docs/端到端调试.md §5.1`） | 升级到 **4.4+**（本工程已用 4.4.1，此 bug 已修）。临时救已卡死的进程：`pkill -9 -f Godot` |
| 点 GUT 报错堆栈时提示打不开外部编辑器 | Godot 的 `Text Editor > External > Exec Path` 为空或路径不可执行 | 设成 `command -v code` 的绝对路径, 参数保持 `{project} --goto {file}:{line}:{col}` |
| 断点不停 | VS Code 没连上 Godot LSP | 重启 Godot，再 `Open workspace with Godot editor` |
| 改完 `.tres` 编辑器没刷新 | Godot 资源缓存 | `Project → Reload Current Project` |
