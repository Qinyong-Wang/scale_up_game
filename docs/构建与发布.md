## 2. 前置条件

| 依赖 | 用途 | 安装/获取 |
|---|---|---|
| Godot 4.4.1 引擎 | 编辑器 + CLI 导出 | 见 `开发环境配置.md` §2.1 |

| 工程能正常打开 | 资源已 import | `godot --headless --import` |
| 测试回归通过 | 防止打出来的包是坏的 | 单元测试干净；集成测试退出码为 0，见 `开发环境配置.md` §3.1 |

> **导出模板放哪**：macOS 默认 `~/Library/Application Support/Godot/export_templates/4.4.1.stable/`。Godot 自己会找，不用手动配。

## 3. 版本号约定

### 3.1 格式

`MAJOR.MINOR.PATCH[-TAG]`，遵循 SemVer：

- `MAJOR` — 不兼容存档 / 不兼容 API
- `MINOR` — 新增系统 / 新功能，存档兼容
- `PATCH` — bug 修复 / 平衡性数值
- `TAG` — `alpha` / `beta` / `rc.1` 之类，正式版省略

例：`0.1.1-alpha`、`0.3.2`、`1.0.0-rc.1`。

### 3.2 单一来源

版本号定义在 **`project.godot`** 的 `[application]` 段：

```ini
[application]

config/name="Scaling Up"
config/version="0.1.1-alpha"   ; ← 唯一 source of truth
```

游戏内运行时这样取：

```gdscript
ProjectSettings.get_setting("application/config/version", "0.0.0-dev")
```

> **不要**在脚本里硬编码版本号，也不要写在多个 `.cfg` 里。改版本号 = 改 `project.godot` 一处。

### 3.3 何时 bump

| 操作 | 时机 |
|---|---|
| 改 `config/version` | 准备打包前的最后一个 commit |
| 打 git tag `v0.1.1-alpha` | 包确认产出且烟雾测过之后 |
| 写 `CHANGELOG.md`（待建） | 跟 bump 同 commit；现在没文件，先用 commit message 凑合 |

## 4. 配置导出预设

### 4.1 创建 `export_presets.cfg`

`.gitignore` 里 `export_presets.cfg` 是被忽略的（里面可能含本地路径 / 证书），**每台开发机要自己生成**：

1. Godot 编辑器 → `Project → Export...`
2. `Add... → macOS`
3. macOS 关键字段：
   - **Application → Bundle Identifier**：`com.scalingup.game`（或你的反向域名）
   - **Application → Version**：优先留空，让游戏内版本仍以 `config/version` 为准；如果 Godot 写入固定值，打包前必须同步到当前数字基准版本
   - **Application → Short Version**：同上；macOS 通常只接受 `MAJOR.MINOR.PATCH`，例如 `0.1.1-alpha` 对应 `0.1.1`
   - **Application → Icon**：`res://icon.icns`（macOS 原生图标；不要留空只靠 `icon.svg` 自动转换）
   - **Display → High Res**：✅
   - **Codesign → Codesign**：选 `Disabled`（首发先不签名，见 §5）
4. `Add... → Windows Desktop`
5. Windows 关键字段：
   - **Application → Modify Resources**：✅（否则不会把图标 / 版本资源写进 `.exe`）
   - **Application → Icon**：`res://icon.ico`（Windows 原生图标；不要留空）
   - **Application → File Version / Product Version**：同步到当前数字基准版本，例如 `0.1.1`
6. 不要点 Export，关掉对话框 — 文件已经写到 `export_presets.cfg`

`README.md` 里的 CLI 构建命令默认这个本地文件已经存在。`export_presets.cfg` 继续保持 gitignored，不要提交到仓库。

### 4.2 检查产出

```bash
ls export_presets.cfg   # 应该存在
grep '^name="macOS"$' export_presets.cfg
grep 'application/icon="res://icon.icns"' export_presets.cfg
grep 'export_path="build/macos/Scaling-Up.app"' export_presets.cfg
grep '^name="Windows Desktop"$' export_presets.cfg
grep 'application/modify_resources=true' export_presets.cfg
grep 'application/icon="res://icon.ico"' export_presets.cfg
grep 'export_path="build/windows/Scaling-Up.exe"' export_presets.cfg
```

如果 `application/version` 或 `application/short_version` 不是空字符串，确认它们与 `project.godot` 的数字基准版本同步；例如 `config/version="0.1.1-alpha"` 时，macOS 字段应为 `0.1.1`。

## 5. 打包流程（macOS）

### 5.1 一次性手动流程

```bash
# 0. 在干净工作树上打包
git status   # 应该 clean

# 1. 跑测试回归
godot --headless --path . -s addons/gut/gut_cmdln.gd -gdir=res://tests/unit -gexit
# 单元测试应无失败、无 pending、无 orphan

godot --headless --path . -s addons/gut/gut_cmdln.gd -gdir=res://tests/integration -gexit
# 集成测试退出码必须是 0；当前可能仍打印少量已知 pending/risky/orphan 诊断

# 2. 准备产出目录
mkdir -p build/macos

# 3. 调用 Godot CLI 导出
godot --headless --path . --export-release "macOS" build/macos/Scaling-Up.app
# 第一次跑会很慢（要拷贝模板），之后增量

# 4. 烟雾测试：双击打开
open build/macos/Scaling-Up.app
```

> **未签名 `.app` 给别人**：第一次打开会被 Gatekeeper 拦。Readme 里要写"右键 → 打开 → 仍然打开"的绕过步骤。要彻底避开这个，得走 §5.3 的签名 + 公证。

### 5.1bis 导出包资源加载注意

Godot 导出后，`.tres` 等资源可能在包内以 `.tres.remap` 形式出现；发布包里 `res://` 目录枚举也不稳定。凡是运行时需要加载一批静态资源的系统（例如数据集市场、拍卖行收藏品、NPC 公司表），不要依赖：

```gdscript
DirAccess.open("res://resources/data/...").list_dir_begin()
```

发布安全的做法是维护显式 `{id: path}` 路径表，用 `load(path)` / `ResourceLoader.exists(path)` 加载，并在单测里检查“路径表与磁盘 `.tres` 一致”。新增或删除数据 `.tres` 时，同步更新对应路径表；否则编辑器里正常，编译版可能出现列表为空。

### 5.2 打 zip 给别人

```bash
cd build/macos
VERSION=$(awk -F= '/^config\/version=/{gsub(/"/, "", $2); print $2}' ../../project.godot)
zip -r -y "Scaling-Up-${VERSION}.zip" Scaling-Up.app
```

`-y` 保留符号链接（macOS .app 内有），不能省。

### 5.2bis Windows 打包流程

```bash
mkdir -p build/windows
godot --headless --path . --export-release "Windows Desktop" build/windows/Scaling-Up.exe
cd build/windows
VERSION=$(awk -F= '/^config\/version=/{gsub(/"/, "", $2); print $2}' ../../project.godot)
zip -r -9 "Scaling-Up-${VERSION}-windows-x86_64.zip" Scaling-Up.exe Scaling-Up.pck
```

Windows 导出必须同时分发 `Scaling-Up.exe` 与 `Scaling-Up.pck`。二者要放在同一目录；只发 `.exe` 会启动失败。若 `.exe` 图标为空，优先检查本地 `export_presets.cfg` 里的 `application/modify_resources=true` 和 `application/icon="res://icon.ico"`。

### 5.3 签名 + 公证（暂缓）

正式发行前要做，否则用户体验差。需要：

- Apple Developer 账号（$99/年）
- `Developer ID Application` 证书（导出 .p12）
- `xcrun notarytool` + App-Specific Password

## 7. 构建产物目录约定

```
build/                  ← 已 gitignored
  macos/
    Scaling-Up.app
    Scaling-Up-<version>.zip
  windows/
    Scaling-Up.exe
    Scaling-Up.pck
    Scaling-Up-<version>-windows-x86_64.zip
exports/                ← 已 gitignored；预留给 dev 调试导出
```

> **不要**把 `build/` 里的东西迁到工程里。它就是临时产物，删了能重建。

## 8. 发布前检查清单

打正式包前必须勾完：

- [ ] `git status` 干净
- [ ] `project.godot` 的 `config/version` 已 bump
- [ ] 本地 `export_presets.cfg` 存在，且版本字段为空或与 `project.godot` 的数字基准版本同步
- [ ] 单元测试 `godot --headless ... -gdir=res://tests/unit -gexit` 无失败、无 pending、无 orphan
- [ ] 集成测试 `godot --headless ... -gdir=res://tests/integration -gexit` 退出码 0
- [ ] `res://icon.icns` 存在，且 macOS preset 的 `application/icon` 指向它
- [ ] `res://icon.ico` 存在，且 Windows preset 的 `application/modify_resources=true`、`application/icon` 指向它
- [ ] 资源路径表一致性测试通过（如 `DatasetSystem.*_PATHS` / `CollectionSystem.COLLECTIBLE_PATHS`）
- [ ] 启动游戏跑过一个完整回合（手测烟雾流程）
- [ ] 检查 `Log` 输出无 `WARN` / `ERROR`（见 `design/开发调试设计.md`）
- [ ] 包大小合理（当前 macOS 包约 260–300 MB；明显超过 350 MB 时检查是否混入缓存或未压缩素材）
- [ ] 双击 `.app` 在另一台干净 macOS 上能启动

## 9. 常见问题

| 现象 | 原因 | 处理 |
|---|---|---|
| `error: No export template found` | 没装 4.4.1 macOS 导出模板 | Godot → `Manage Export Templates → Download and Install` |
| 导出后 `.app` 一打开就闪退 | 主场景配错 / autoload 报错 | 终端跑 `./Scaling-Up.app/Contents/MacOS/Scaling\ Up` 看 stderr |
| `.app` 打不开（"无法验证开发者"） | 没签名 + Gatekeeper | 临时绕：右键 → 打开；长期：走 §5.3 签名 |
| 版本号不对 | export 配置硬写了版本，没读 `config/version` | 编辑 `export_presets.cfg`，把 `application/version` / `application/short_version` 留空或同步到当前数字基准版本 |
| `.app` 图标显示为空白 / 通用白纸 | macOS preset 没指定原生 `.icns`，或 Finder 缓存了旧 bundle 图标 | 确认 `application/icon="res://icon.icns"` 后重新导出；必要时换输出目录或刷新 Finder 图标缓存 |
| 编译后某个目录型内容为空（拍卖行 / 数据集市场等） | 代码依赖 `DirAccess` 枚举 `res://`，或只匹配 `.tres` 但导出包内是 `.tres.remap` | 改成显式 `{id: path}` 路径表加载，并加“路径表与磁盘一致”单测；重打包后烟雾测试 |
| zip 体积过大 | `.godot/` 缓存或调试 import 一起进了 | 重打前 `rm -rf build/`，确认 `.gitignore` 中 `.godot/` 生效 |
