# 宇宙模拟工程设计（慈善三期 capstone）

**一句话目的**：慈善系统的收尾大工程——一条 5 级递进的「模拟」阶梯（气象 → 海洋 → 地球 → 太阳系 → 宇宙）。每一级都要玩家**真的选一座自有数据中心永久捐出去**（达到该级的 FLOPs 门槛）**+ 一大笔捐助资金 + 更长的计算时间**；逐级解锁，跑完最后一级「宇宙模拟」时揭晓终极答案 **42**，并在办公室摆出可点击打开的「终极答案盒」。

> 这是 `慈善系统设计.md §10` 规划的「三期」。复用 `TaskSystem`（每一级是一个长任务）与二期已埋好的 `universe_answer` 奖杯（`办公室与收藏系统设计.md §4`）。

## 1. 定位与玩法

- **后期资金 + 算力的终极出口**：纯收尾 / 炫耀内容，不给战力加成。它要求玩家既有**巨额现金**又有**真实的算力帝国**，是把游戏玩到头的标志。
- **逐级阶梯**：5 级按固定顺序解锁；第 N 级需第 N-1 级已完成。
- **每级三道门槛递增**：
  1. **算力门槛 `min_train_tflops`**：玩家要**捐出一座自有（owned）、空闲（idle）、未开启对外出租（`rent_out_enabled == false`）的数据中心**，其训练算力 `train_tflops`（已含 GPU 单卡算力 × 卡数 × 集群效率 × 生态 × 太空加速的真实派生值）必须 ≥ 该级门槛。**按单座 DC 的 FLOPs 计，不是 GPU 总数**——逼玩家堆出一座真正的大算力中心，而不是凑一堆小机房。正在出租到算力平台的 idle DC 视为已有对外承诺，必须先停止出租，不能直接永久捐出。
  2. **资金 `cost`**：启动该级时一次性捐助的现金（递增，宇宙级 ~¥1T）。
  3. **时间 `weeks`**：该级模拟计算要跑的周数（递增，宇宙级最长 ~52 周）。
- **真的选 + 永久消耗**：启动该级要弹出对话框（§8），玩家从满足门槛的自有、空闲、未出租 DC 中**选一座**捐出。确认后该 DC **从 `GameState.datacenters` 永久移除**（连卡带机房一起捐走，不退款、不返还），同时一次性扣 `cost`，再起任务跑 `weeks` 周。算力中心是「捐建给科研」的——一去不回。
- **启动即扣资金 + 消耗 DC、跑满才完成**：和慈善一样是 `TaskSystem` 长任务（启动当周扣 `cost` + 移除 DC、跑 `weeks` 周、完成才推进）。资金按「捐助」处理，**可抵税**（reason `simulation_funding` 不在 `NON_TAXABLE_REASONS`）。
- **终局规模锚点**：宇宙级门槛标定为「`facility_planet`（100M 卡微型星球）装满最好 GPU（Cypress T3）」这一座 DC 的训练算力（grid 供电下 ~1.94×10¹¹ TFLOPs）；门槛取 1.5×10¹¹ TFLOPs，只有这种体量的单座 DC 能清门槛。加上 ~¥1T 捐助现金 + 自建微型星球本身的天价，正是「卷到头」的终局目标。下面 4 级按机房档位往回推（见 §9）。
- **终局**：跑完「宇宙模拟」→ emit `universe_answer_revealed`、把 `universe_answer` 追加进 `GameState.trophies` 并 emit `trophy_awarded`（办公室出现「终极答案盒」）、UI 弹窗提示玩家去办公室查看；玩家点击盒子后在荣誉信息 dialog 里揭晓「42」。

## 2. 阶段数据模型

数值放 `resources/data/simulation/*.tres`（`SimulationStageSpec` 资源），`SimulationSystem` 在 `_ready` 按 `order` 加载。

`SimulationStageSpec`（`scripts/resources/simulation_stage_spec.gd`）：

- `id: StringName` — 阶段 id（`&"weather"` / `&"ocean"` / `&"earth"` / `&"solar_system"` / `&"universe"`）。
- `order: int` — 阶梯顺序（0..4）。
- `display_name: String` / `description: String` — UI 文案（走 i18n）。
- `cost: int` — 启动该级一次性捐助资金。
- `weeks: int` — 计算周数。
- `min_train_tflops: float` — 启动门槛：要捐出的那座自有、空闲、未出租 DC 的训练算力 `train_tflops` 下限。

## 3. 阶梯状态与流程

- `GameState.simulation_stages_done: int` — 已完成的阶段数（0=未开始，5=全部完成）。下一可启动阶段 index = `simulation_stages_done`（若 < 5 且当前无 simulation 任务在跑）。
- **启动**：`simulation.start_stage {dc_id}`（`dc_id` = 玩家选中要捐出的数据中心）。校验：
  - 有下一可启动阶段（`simulation_stages_done < 5`），否则 `&"all_done"`；
  - 当前无 `simulation` 子类型任务在跑（一次只跑一级），否则 `&"already_running"`；
  - `dc_id` 存在于 `GameState.datacenters`，否则 `&"unknown_dc"`；
  - 该 DC 为自有（`ownership == &"owned"`），否则 `&"dc_not_owned"`（租用 / 云租的算力不归你所有，不能捐）；
  - 该 DC 空闲（`status == &"idle"`），否则 `&"dc_busy"`（训练 / 部署中不能捐）；
  - 该 DC 未开启对外出租（`rent_out_enabled == false`），否则 `&"dc_rented_out"`（先停租再捐）；
  - 该 DC 的 `train_tflops ≥ 该级 min_train_tflops`，否则 `&"compute_too_small"`；
  - `cash ≥ cost`，否则 `&"insufficient_cash"`（不允许捐成负数）。
  通过 → **先把该 DC 从 `GameState.datacenters` 永久移除**（emit `datacenter_removed`，不退 GPU 残值），再调 `task.start({template_id: &"simulation_stage", stage_id, amount: cost, weeks})`，启动当周扣 `cost`（reason `simulation_funding`）。
- **完成**：任务完成 → `TaskSystem` 走 `completion_command = &"simulation.complete_stage"`，payload `{stage_id}`。`SimulationSystem._on_complete_stage`：`simulation_stages_done += 1`；emit `simulation_stage_completed(stage_id, simulation_stages_done)`；若刚完成的是最后一级（`universe`）→ 追加 `universe_answer` 进 `trophies` + emit `trophy_awarded` + emit `universe_answer_revealed`。UI 收到 `universe_answer_revealed` 时只弹一次提示，确认后切到办公室；真正的「42」仍由玩家点击办公室里的盒子打开。

## 5. 任务接入（复用 TaskSystem）

`simulation` 作为 `TaskSystem` 新子类型接入，复用慈善的 `charity_law` 动态时长/费用（payload 读 `weeks`/`amount`）。改动面：

- 模板 `resources/data/tasks/simulation/simulation_stage.tres`（`subtype = &"simulation"`，`duration_func = &"charity_law"`）。
- `_resolve_completion_command(&"simulation")` → `&"simulation.complete_stage"`。
- `_resolve_completion_payload`（simulation 分支）→ `{stage_id}`。
- 启动扣费 reason：subtype == `simulation` → `&"simulation_funding"`（可抵税，单列账本类目「宇宙模拟」）。
- **不锁 DC，而是直接消耗**：启动时 `SimulationSystem` 把选中的 DC 永久移出 `GameState.datacenters`（见 §3），任务本身不再持有 / 锁定算力。任务取消（`task.cancel`）只按既有规则退部分现金，**被捐出的 DC 不返还**。

## 9. 阶梯参考数值（仅参考，权威在 `.tres`）

门槛按「机房档位装满最好 GPU（Cypress T3，2500 TFLOPs/卡，生态 1.0）」往回推，门槛取该档真实 `train_tflops`（grid 供电、无总工程师）的 ~65~80% 留余量，使得**恰好这一档体量的单座 DC 能清门槛、低一档清不了**：

| 阶段 | order | 对标机房（装满 Cypress T3）| `min_train_tflops` | 可读门槛 | 时间 `weeks` | 资金 `cost` |
|---|---|---|---|---|---|---|
| 气象模拟 weather | 0 | 16k 卡建筑 (building_s，实算 ~3.2e7) | 2.0e7 | 20 PFLOPs | 8 | 1B |
| 海洋模拟 ocean | 1 | 100k 卡建筑 (building_l，实算 ~1.9e8) | 1.2e8 | 120 PFLOPs | 16 | 10B |
| 地球模拟 earth | 2 | 100 万卡园区 (campus_l，实算 ~1.8e9) | 1.2e9 | 1.2 EFLOPs | 26 | 50B |
| 太阳系模拟 solar_system | 3 | 2000 万卡太空 (space_l，实算 ~4.0e10) | 2.5e10 | 25 EFLOPs | 39 | 200B |
| 宇宙模拟 universe | 4 | **1 亿卡微型星球 (planet，实算 ~1.94e11)** | 1.5e11 | 150 EFLOPs | 52 | 1T |

> 资金逐级 ×5~10、算力门槛逐级翻一两个数量级；宇宙级要求一座装满最好 GPU 的微型星球（光自建就 ~¥7.4T、含卡）+ ¥1T 捐助现金 + 一年计算——把游戏卷到头的终局。`train_tflops` = 单卡 TFLOPs × 卡数 × 集群效率(含大集群衰减) × 生态评分 × (1+太空加速)。具体数值以 `.tres` 为准，可继续按后期真实量级调平衡。
