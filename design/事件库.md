## 1. 四类事件

每张卡片是 `resources/data/events/<id>.tres`, schema 见 `scripts/resources/event_card.gd` 与 [事件系统设计.md §1](事件系统设计.md)。`category` 决定玩法语义:

- **`opportunity`** — 玩家选择拿好处 / 拒绝, 不会负面。
- **`crisis`** — 玩家通常要花钱减伤, 拒绝可能更糟 (扣订阅 / 关停资产)。
- **`flavor`** — 单纯叙事, 没有选项, 玩家"知道了"即关。
- **`routine`** — 每 12 周强制弹一张, 必须选 option 才能进下一周; 内容是公司经营日常的无厘头小决策。

**所有卡都至少满足一个前置条件** (v10 约定): 事件要贴合当前局势, 不能凭空乱弹。条件字段见 [事件系统设计.md §1](事件系统设计.md)。routine 池里至少 5 张只 gate `min_turn >= 4` 且使用默认 3 次硬上限, 保证停滞局也有足够轮换。

`dc_terminate` / `dataset_delete` 类 effect 不带 id 时由 EventSystem 随机挑一个目标; `product_boost_subscribers` 不带 id 时挑订阅数最多的产品。

因此, 凡是 option 或 passive effect 里出现 `product_boost_subscribers`, 卡片本身必须写 `requires_product=true`。`economy_award` 只用于真实外部现金流入, 或明确由具体工具/外包方案带来的可落地成本节省; "把预算省下来"、"保持诚实"、"占领道德高地"这类抽象姿态不能翻译成直接加钱, 否则玩家会觉得系统在无中生有发钱。

破坏性 effect 的 option 可以保持戏剧性, 但按钮后果预览必须承担规则说明: `dc_terminate` 要写清关停会移除数据中心, 且自购 GPU 会先二手出售; `dataset_delete` 要写清永久删除。若文案写到"招来研究员 / 收编团队 / lead 离开"等资产层面的变化, 对应系统必须真的有 effect 支撑; 否则只能写成顾问背书、团队声誉、市场信心、订阅波动等软后果。

`dc_meltdown` 是永久资产损失类危机, 需额外遵守首年保护 + 限次: `min_turn=52`, `max_triggers=2`。`routine_open_source_pr` 是高频轻量 routine, 也设 `max_triggers=2`。

## 1bis. drama 真两难卡 (v11)

一批"取材自 AI 行业真实争议 + 硅谷创业喜剧梗"的卡, 全部**化名/泛指 + 转化改写** (不出现真实公司 / 人名 / 品牌, 不照搬剧集原文; 仅在 `build_event_cards.py` 注释里用 `≈`/"原创演绎"标注灵感来源)。两个选项都挂真生效 effect, 形成对立取舍; 多数 `max_triggers=1`。卡片数值仍以 `.tres` 为准。

**AI 历史争议线 (6 张)**: `board_coup` (≈ 创始人被董事会闪电解雇又复职) · `pause_letter` (≈ 千人联名要求暂停训练 6 个月) · `data_lawsuit` (≈ 出版巨头起诉违规抓取训练数据) · `sentient_engineer` (≈ 工程师公开宣称模型有意识) · `celebrity_voice` (≈ 影星指控语音助手盗用其声线) · `doomer_vs_acc` (≈ 公司内部"末日派"对"加速派"互撕)。 (2026-05 删除原线中的 `moat_memo_leak` / `weights_leak` / `deepseek_moment` 三张"开源/降价"主题卡, 见顶部删卡说明。)

**硅谷创业喜剧线 (11 张, 均转化改写)**: `three_commas_investor` (炫富暴发户投资人乱指挥) · `middle_out` (天才工程师的"颠覆性压缩黑科技"神话) · `not_hotdog` ("这是不是煎饼果子"无脑分类 App 爆火) · `hooli_keynote` (搜索巨头发布会抄你产品, `max_triggers=2`) · `platform_pivot` (投资人逼你"别做产品要做平台") · `rogue_agent` (内部编程 agent 半夜自作主张狂开 GPU) · `benchmark_gaming` (全网信仰的跑分榜与刷榜诱惑) · `hardware_box_pivot` (空降职业经理人主张"别做云改卖盒子") · `exclusive_megadeal` (天价独家大单的锁死) · `rebrand_consultant` (网红品牌顾问的"性感重塑"豪赌) · `fake_users` (融资前夜撞破刷量冲数据)。

**非技术向纯喜剧 (2 张, 只玩钱和团队、不碰订阅/技术)**: `ai_orders_beef` (AI 助理擅自订购三吨牛肉) · `doomsday_bunker` (投资人坚持先挖末日地堡)。

**灰暗 / 伦理向 (4 张, 点到为止的社会讽刺, 利润 vs 良心)**: `labeling_sweatshop` (≈ "血奴"/数据标注血汗工厂) · `surveillance_contract` (≈ 国防/大规模监控合同) · `companion_tragedy` (≈ 陪伴型 AI 为留存把脆弱用户越拉越深) · `crunch_culture` (≈ 996/过劳累垮)。这批卡刻意让"黑暗选项"在机制上有诱惑 (省钱/涨数据), 逼玩家真做取舍; 文案点到为止, 不猎奇。2026-05: 玩家反馈这类"血奴/血汗"卡几乎刷不到 (落在庞大随机池、门槛偏高), 故给 `crunch_culture` / `labeling_sweatshop` 放宽前置 (min_turn 24→12 / 30→16, crunch 员工门槛 2→1) 并把 `weight` 8→24, 让它们更容易被抽到; 仍保留 `max_triggers=1`。

**AI 行业黑色幽默一次性卡 (6 张, v17)**: `support_bot_refund_policy` (客服机器人编出退款政策) · `forum_wisdom_summary` (检索增强把论坛玩笑写成操作建议) · `fictional_case_law` (法务材料里出现不存在的判例) · `history_image_overfit` (历史图片生成被安全模板过拟合) · `support_bot_self_roast` (客服机器人公开吐槽自家产品) · `compliance_bot_illegal_advice` (合规助手一本正经教人违规)。这批全部 `max_triggers=1`, 不使用真实公司 / 人名 / 品牌; 只把现实事件抽象成化名世界里的经营事故。

### legacy / 时间锚点卡

- ~~`funding_offer` (opportunity): 早期"破产逃生"小额融资~~ **2026-05 删除** (老的"天使投资人主动联系", 与新 8 轮顺序融资体系 [经济系统设计.md §4.6](经济系统设计.md) 重复)。其在测试里"唯一可在 turn=1 空状态抽中的金丝雀卡"角色由 `debug_test_offer` (`category=debug`, `weight=0`, 不进实战随机池) 接替。
- 时间锚点·技术范式卡 (`paradigm_rlhf / moe / long_ctx / reasoning_rl`): `weight=0` 不进随机池, 在固定 `min_turn` 由 `_maybe_trigger_paradigm_events()` 确定性触发, 一次性。**v8 PR-H 起为纯叙事卡**, 不再触碰 NPC capability, 也不再保留隐藏 option。

## 1ter. 历史档案事件 (v12)

历史档案事件的目的不是制造选择压力, 而是在经营过程中给玩家逐步补齐 AI 史脉络。全部为无选择 `flavor` 卡:

- 只显示标题、正文与"知道了"按钮; 玩家无需选项。
- 首批历史档案没有 `passive_effects`: 只承担知识提示和时间线氛围, 不承担经济平衡功能。
- `weight=0 + max_triggers=1 + cooldown_months=9999`, 由 `EventSystem.HISTORICAL_EVENTS` 按历史顺序触发。
- UI 文案只写泛化/化名表达; 现实锚点可在本设计文档中记录。

当前首批 13 张覆盖 2017-2025 的主干节点。1950-2016 的前史不作为游戏内回合事件, 否则会和 2017 开局时间线冲突; 后续可放进帮助/教程的"AI 前史"条目。

## 2. 加事件的步骤

1. 在 `tools/build_event_cards.py` 的 `CARDS` 表加一条 (或手写 `.tres`), 跑脚本生成。
2. 在 `EventSystem.EVENTS` 字典加 entry。
3. 如果用了新 `effect.kind`, 在 `EventSystem._apply_effect` 派发表加 case + 同步 [公共枚举表 §12](公共枚举表.md)。
4. **每张卡至少 gate 一个前置条件** (v10 约定)。
5. 写 unit test (参考 `tests/unit/event_system_test.gd`)。

## 3. 未决问题

- [ ] **持续效果**: 没有 `BuffSystem`, effect 都是瞬时 (含比例化瞬时)。"泡沫期用户 ×1.5 持续 6 月"无法表达。
- [ ] **事件并发**: FIFO 一次一个。
- [ ] **`dc_terminate` 在 busy dc 上失败**: InfraSystem 要求 idle, 强行 terminate 会 applied 记 error (不崩溃)。
- [ ] **回滚**: 玩家选错不能撤销。
