# 音频系统设计

> **一句话目的**: 给游戏配一层纯乐器背景音乐 (轻松电子科技 / 原声有机 / 磅礴混合) 与低打扰 UI 点击音效; BGM 由 Replicate MusicGen 离线生成入库, SFX 由运行时代码生成短样本。

## 1. 范围

当前实现两类:
- **背景音乐 (BGM)**: 23 首纯乐器曲目 (电子科技 / 原声有机 / 磅礴混合三类), 打乱后顺序循环播放。
- **界面音效 (SFX)**: 先覆盖按钮点击。回合推进 / 通知音仍留作未决问题 (§7)。

## 2. 组成

| 件 | 位置 | 职责 |
|---|---|---|
| `MusicPlayer` (autoload) | `scripts/autoload/music_player.gd` | 持有一个 `AudioStreamPlayer`, 启动打乱 23 首曲序后**顺序循环**播放; 起 / 停**渐入渐出**; 受 `Preferences.music_enabled` 控制开关。 |
| `Preferences.music_enabled` | `scripts/autoload/preferences.gd` | 开关状态, 落盘到 `user://preferences.cfg` 的 `[audio]` 段 (与 locale 同文件不同段)。 |
| `SfxPlayer` (autoload) | `scripts/autoload/sfx_player.gd` | 持有一组短音效播放器; 自动注册进入场景树的 `BaseButton`, 点击时播放短促 click, 同时做轻微按压缩放。 |
| `Preferences.sfx_enabled` | `scripts/autoload/preferences.gd` | 界面音效开关, 同样落 `[audio]` 段。 |
| 设置开关 | `scenes/ui/settings_dialog/settings_dialog.gd` | 「背景音乐」CheckButton + 音乐音量滑块; 「界面音效」CheckButton; 起始页与游戏内共用同一对话框。 |
| 曲目资产 | `assets/audio/music/*.mp3` | 纯乐器 BGM, 每首约 3~4 分钟, 由 `tools/generate_music.py` 生成 (Godot 4.3 原生支持 .mp3)。 |

## 3. 播放行为

- `MusicPlayer` 在 `_ready` 时把 `TRACKS` 复制到 `_playlist` 并 `shuffle()` (非测试运行); 若 `Preferences.music_enabled` 为真则起播 (渐入)。
- `_playlist` 曲目**顺序播放**, 单曲 `finished` → 切下一首; 到末尾回头第一首 (歌单循环)。23 首打乱轮换, 一轮约 1.3 小时才重复。
- 测试运行下不打乱 (保持原序), 让下标推进断言确定。
- 单曲**不开 loop**, 循环由播放器在 `finished` 上推进, 否则 `finished` 永不触发。
- 播放目标音量由 `Preferences.music_volume`(线性 0..1) 决定: `_target_db()` = `linear_to_db(clamp(music_volume, ε, 1))`, 并 clamp 到 `[SILENT_DB, MAX_VOLUME_DB]`; `music_volume≈0` 视作静音 (回 `SILENT_DB`)。`MAX_VOLUME_DB`(默认 -2 dB) 让满档也不抢前景, 默认 `music_volume=0.5` ≈ -6 dB (≈ 旧固定 -8 dB 档)。`VOLUME_DB` 常量仅留作参考, 实际播放走 `_target_db()`。
- 缺曲目 (文件没导入 / 路径错) 时静默早退并 `Log.warn`, 不崩。

### 3.1 渐入渐出

- 起播 / 续播: 音量从 `SILENT_DB`(-40 dB) 用 `Tween` 渐入到 `_target_db()`, 时长 `FADE_SEC`。
- 停播 (关开关): 渐出到 `SILENT_DB` 后再 `stop()`。
- 曲目文件本身也在生成时烘焙了首尾淡化, 故**曲目自然切换**也平滑 (无需播放器额外处理接缝)。
- **测试运行下跳过 Tween**, 起 / 停同步发生 (Tween 需逐帧推进, headless 测试里不可靠), 由 `_use_fades()`=`not _is_test_run()` 控制。

## 4. 开关与持久化

- BGM 偏好**只有一个消费者** (`MusicPlayer`), 故不走 `EventBus` 广播 (对比 locale: 多个 UI 要重渲染才广播)。设置对话框直接调 `MusicPlayer.set_enabled(on)`。
- `MusicPlayer.set_enabled(on)`: 写 `Preferences.set_music_enabled(on)` 持久化 + 立即起播 / 停播。
- `MusicPlayer.set_volume(v)`: 写 `Preferences.set_music_volume(clamp(v,0,1))` 持久化 + 若在播则把 `_player.volume_db` 调到新 `_target_db()` (测试运行下同步设, 否则渐变)。开关 (`music_enabled`) 与音量 (`music_volume`) 正交: 开关管起 / 停, 音量管响度。
- `SfxPlayer.set_enabled(on)`: 写 `Preferences.set_sfx_enabled(on)`; 关闭后仍保留按钮视觉反馈, 只静音 click。
- `Preferences.set_music_enabled(v)` / `set_music_volume(v)` / `set_sfx_enabled(v)`: 值未变早退; 变了写盘 (`[audio] music_enabled` / `music_volume` / `sfx_enabled`), 不全量覆盖 cfg (保留 `[i18n]`)。
- 下次启动 `Preferences._load()` 读回 → `MusicPlayer._ready` 据此决定是否起播。
- **测试 hermetic**: 测试运行 (`-gtest` / `gut_cmdln`) 下 `MusicPlayer._ready` 不自动起播 (镜像 `Preferences._is_test_run`), 避免污染。

## 4bis. 按钮点击 SFX

- `SfxPlayer` 不依赖静态音频文件: `_make_button_click_stream()` 运行时生成约 55ms 的 `AudioStreamWAV`, 双高频正弦叠加快速衰减, 音色像轻触控制台按钮, 避免抢 BGM。
- 为避免连续点击截断, `SfxPlayer` 内部维护 4 个 `AudioStreamPlayer` 轮询播放; 全部走 `Master` bus, 当前不拆独立 SFX bus。
- 自动注册 `BaseButton`:
  - `pressed` → 若 `Preferences.sfx_enabled` 为真, 播放 click。
  - `button_down/button_up/mouse_exited` → 按钮围绕中心做极轻微缩放回弹, 不改变 Container 布局。
  - 注册同时把鼠标形状设为 pointing hand, 让可点目标更明确。
  - 按钮退出场景树时, `SfxPlayer` 必须断开它挂上的所有按钮相关信号并清掉内部跟踪; 若同一个 Button 实例被弹窗 / 容器复用后重新进树, 再次注册必须是干净的一次连接, 不产生 Godot 的重复连接错误。
- 测试运行下也会创建播放器与生成音频流, 但不会自动发声; 单测只断言 stream 生成、注册和开关逻辑。

## 5. 曲目生成 (离线, 开发期)

`tools/generate_music.py` 调 **Replicate 的 MusicGen** (`meta/musicgen`, `stereo-large`) 一次性生成约 3.5 分钟的**连续**纯乐器曲 —— 自回归单段生成, 无拼接, 故**无「切歌」接缝** (这是放弃 Vertex AI Lyria 30s 拼接方案的原因: 拼出来每 ~30s 像换歌)。生成后用 `ffmpeg` 烘焙首尾淡入淡出 + 编码 MP3 (Godot 4.3 原生支持)。

- **鉴权**: 复用美术工具的 `REPLICATE_API_TOKEN` (env 或 `tools/art/.env`)。
- **依赖**: `ffmpeg`/`ffprobe` + `replicate` 库 (装在 `tools/art/.venv`)。**必须用该 venv 的 python 跑** (系统 python 无 replicate; 且 raw urllib 会被 Cloudflare 1010 拦)。
- `meta/musicgen` 是普通模型 (非 official), `run()` 必须 `model:version`, 否则走 official 端点 404。
- **化名规范**: 真实模型名只出现在该生成工具里, **不进**运行时代码 / `.tres` / UI 文案。
- 23 首, 分三类基调, 各自调性 / 速度 / 音色不同; `--force` 才重生成已存在曲目, 否则跳过 (试听后删了 8 首低质电子曲, 编号 `01/02/03/05/07/08/11/12` 留空档不复用):
  - `bgm_04/06/09/10/13/14/15` (7 首) —「轻松电子科技」(合成器 / lo-fi / techno)。
  - `bgm_16`..`bgm_27` (12 首) —「原声有机」(钢琴 / 弦乐 / 原声吉他 / 世界乐器), 温暖柔和, 与电子曲反差。
  - `bgm_28`..`bgm_31` (4 首) —「磅礴混合」(电影感管弦叠合成器), 用于大场面 (宇宙模拟 capstone / 重大里程碑)。
- 并发: 一次提交 5 首 (Replicate 端并行), 单首失败不拖垮整批 (`--only` 重试)。

## 6. 验收

- `MusicPlayer.is_enabled()` 反映 `Preferences.music_enabled`。
- `set_enabled(false)` 停播、`(true)` 起播; 状态写回 `Preferences` 并往返持久化。
- `_on_track_finished()` 把曲目下标 `(i+1) % N` 推进、循环回头。
- 设置对话框有「背景音乐」开关, 切换写 `MusicPlayer`, 文案走 `tr("SETTINGS_MUSIC")` 且随 locale 实时刷新。
- 设置对话框有「音乐音量」滑块 (0..1), 拖动写 `MusicPlayer.set_volume()` 并持久化往返; `_target_db()` 随之变。
- 设置对话框有「界面音效」开关, 切换写 `SfxPlayer.set_enabled()`, 文案走 `tr("SETTINGS_SFX")` 且随 locale 实时刷新。
- `SfxPlayer.register_button(button)` 会连接 `pressed`, 设置手型光标; `pressed.emit()` 在开关开启时推进 click 播放计数, 关闭时不播放。
- `SfxPlayer` 在按钮退出场景树时断开 `pressed/button_down/button_up/mouse_exited/tree_exiting` 连接; 复用同一按钮重新注册不会出现重复连接错误。
