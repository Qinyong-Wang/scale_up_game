# 教程与帮助系统设计

> **一句话目的**：让新玩家开局看懂"训练模型 → 做产品 → 赚钱再投入"的核心循环，并在游戏内随时能查到每个系统怎么玩。

分两块、互不耦合：

1. **开局新手引导** —— 新游戏开始时弹一个**多页分步**对话框，按页讲清核心玩法流程；可勾「不再显示」。
2. **帮助视图** —— 右侧导航「其他」组里放一个「帮助」入口，点进去是一个 master-detail 视图，左侧列系统、右侧看说明；顶部可一键**重新查看新手引导**。

两块共用同一份引导对话框（`TutorialDialog`），帮助视图只是再次把它弹出来。

---

## 1. 开局新手引导 `TutorialDialog`

`scenes/ui/tutorial_dialog/tutorial_dialog.gd`（`extends AcceptDialog`）。

- **多页**：内部维护 `pages` 数组，每页 `{title_key, body_key}`，正文用 `\n` 换行（i18n cell 内字面 `\n`，由 `build_translations.gd` 转真实换行）。
- 页序（核心循环优先）：
  1. **欢迎 / 回合制** —— 你是一家 AI 模型公司的创始人，目标是把模型做强、做出产品赚钱；游戏每周一回合，点顶栏「推进回合」结算。
  2. **训练模型流程** —— 招人 → 建机房买算力 → 备数据 → 在「模型」开训练任务 → 推进回合得到模型 → 评估后才知道能力。
  3. **创建与影响产品流程** —— 用已发布模型在「产品」创建产品 → 产品吸引用户、用户带来营收 → 用营收继续投人 / 算力 / 研究，滚起飞轮；营销、定价、科技都会影响产品表现。
  4. **随时查看帮助** —— 右侧导航的「帮助」里有每个系统的详细说明，迷路了就去看。
- **导航**：`dialog_hide_on_ok = false`；OK 按钮非末页文案为「下一步」、末页为「开始游戏」；额外加一个「上一步」按钮（首页禁用）；显示「第 N / 共 M」页码。
- **不再显示**：底部一个「不再显示」`CheckBox`。对话框**不自己写 Preferences**，而是在真正关闭（末页确认）时 `emit finished(dont_show_again: bool)`，由 `main.gd` 负责落 `Preferences.set_skip_intro(...)` 并 `queue_free`（与 `HonorDialog` "对话框不碰全局态" 同一风格，便于测试）。
- 内容套**定高 `ScrollContainer`**，避免长正文把 OK/上一步按钮顶出窗口（见 memory：ConfirmationDialog 要套 ScrollContainer）。
- **实时切语言**：对话框自身订阅 `EventBus.locale_changed`，开着时切中/英会即时重渲染标题 / 正文 / 上一步 / 不再显示 / OK 文案（对齐 `SettingsDialog` 做法）。关闭时随节点释放自动断开。

### 何时弹

- 新游戏检测靠 `GameState.pending_intro`：一个**会话态布尔、不入存档**（同 `_next_*_seq` 计数器，不进 `to_dict/from_dict`）。
  - 起始页 `start_screen.gd` 确认新游戏时置 `true`；读档 / 继续游戏不置。
  - `main.gd._ready()` 读到后：仅当 `pending_intro && not Preferences.skip_intro && 非截图/自动跑` 时弹出，随后**无条件清零**。
- 截图 / autoplay（`AGI_SCREENSHOT` / `AGI_AUTOPLAY`）模式下**不弹**，保证截图确定性；引导对话框自身的截图走 `_open_drawer_for_screenshot("dialog:tutorial")`。

## 2. 帮助视图 `HelpView`

`scenes/ui/views/help_view/help_view.{tscn,gd}`（`extends VBoxContainer`，与其它 view 一致：不读 `GameState`，`refresh()` 拉内置静态内容即可）。

- **布局**：顶部一行「重新查看新手引导」按钮（`emit replay_tutorial_pressed`）。下面 master-detail：
  - 左：系统**话题列表**（一列按钮，固定宽）。
  - 右：详情面板，标题（粗体）+ `RichTextLabel`（`fit_content`、自动换行、bbcode），由外层 tab 的 `ScrollContainer` 负责溢出滚动（不嵌套滚动）。
- **话题（覆盖全部玩家面向系统，按游玩流程排序）**：回合制 / 模型训练 / 产品与营收 / 营销 / 竞争对手 / 经济与融资 / 招聘与员工 / 基建与算力 / 数据集 / 科技树 / 任务 / 事件 / 慈善 / 收藏与拍卖。
- 每个话题 `{id, title_key, body_key}`，文案全部 `tr()`，落 `strings.csv`（key 前缀 `HELP_`）。
- **i18n**：`refresh()` 里重建按钮文案与当前详情，使语言切换（`_on_locale_changed → _refresh → refresh`）即时生效。

### 接线（`main.gd`）

- `SIDEBAR_NAV` 的 `NAV_GROUP_OTHER` 末尾加 `{"id": &"help", "icon": 0xe887, "label": "NAV_HELP", "tab": "帮助"}`（Material Icons `help`，已验证 `icons.ttf` 含该字形）。
- `_build_tabs` 加 `_make_tab("帮助", "_tab_help")`；`_refresh()` 末尾调 `_render_help_tab()`。
- `_render_help_tab()` 懒实例化 `HelpView`，把 `replay_tutorial_pressed` 接到 `_show_intro_tutorial()`。
- `AGI_INITIAL_NAV=help` 可直接截帮助视图。
