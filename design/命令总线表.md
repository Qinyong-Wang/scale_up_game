# 命令总线表

> **目的**: 集中列出所有 CommandBus 命令名、入参 / 出参 schema、错误码、所有者 system。各系统对外契约总览。
> **读者**: 调用任一命令前先看本表; 增加命令必须先在本表登记。

约定:
- 名字格式 `<domain>.<verb>`; 入参 / 出参皆 `Dictionary` (至少 `ok: bool`, 失败时含 `error: StringName`)。
- 一个命令一个 handler。
- 调用方拿 `r.ok == true` 才用其他字段。
- 时间字段单位统一**周** (1 turn = 1 周)。

| `simulation.complete_stage` | `{stage_id}` | `{ok, stages_done}` | `&"unknown_stage"` (私有, 仅 TaskSystem 完成回调) |

> 宇宙模拟阶梯: 启动下一级要**选一座自有空闲 DC 捐出去** (门槛 = 该 DC 的 `train_tflops` ≥ `min_train_tflops` + 现金 ≥ cost); 通过则**永久移除该 DC**(emit `datacenter_removed`, 不退残值), 起一个 `simulation` 子类型 task; 完成推进阶梯, 跑完宇宙级揭晓 42 + 授 `universe_answer` 奖杯。Per design/宇宙模拟工程设计.md §6。

## 命名约定

- domain 与系统对应 (economy / hiring / infra / dataset / research / tech / task / market / user / product / monetization / marketing / event / charity)。
- verb 用动词原形 (`start`, `cancel`, `add`, `delete`, `lock`, `release`, ...)。
- 私有命令 (`research.add_model` / `product.update_subscribers` 等) 仅供内部 system 调用, UI 不暴露。

## 错误码约定

- `unknown_*`: 找不到目标。
- `*_busy`: 资源被占用。
- `already_*`: 重复操作。
- `not_*`: 不在某状态。
- `*_too_low` / `*_exceeded`: 数值越界。
- `would_*`: 操作会破坏不变量。
- `insufficient_*`: 资源不足 (cash / staff / idle)。

**总线级错误码**: 向未注册的命令 `send` 时, CommandBus 自身返回 `{ok = false, error = &"unknown_command"}` (`command_bus.gd`), 不由任何 handler 产生。正常情况下不应出现, 出现即说明命令名打错或对应 system 未注册。
