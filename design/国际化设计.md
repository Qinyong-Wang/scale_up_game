# 国际化 (i18n) 设计

> **目的**: 给 UI 文案接上 Godot 内置 i18n 管道, 界面支持中文 / 英文实时切换。
> **读者**: 所有写 UI 代码 / 组件 / 对话框 / 抽屉的人。
> **状态**: 🟢 阶段 2 完成 (UI 框架全量 tr 化) — 运行时中/英切换已上线 (见 §11)。strings.csv ~730 key 覆盖: 起始页/设置/侧栏/顶栏 + 全部 10 个 tab view + 全部 12 个对话框 + 4 个 card translator + main.gd (含概览/经济/营收/竞争对手明细行、状态 toast、部署抽屉、specialty/bonus 字典)。
> **故意不翻**: 隐藏 tab 节点名 (tabs_visible=false, 仅做索引匹配); 语言按钮 "中文"/"English" (各以本族文字显示); `DEFAULT_PLAYER_NAME` 默认开局名 (种子数据); `main.gd._sanitize_tech_text` 的科技效果替换表 (渲染 .tres effects_summary, 属内容层)。
> **阶段 3 (游戏内容)**: 🟢 抽取+翻译完成。`tools/extract_content_strings.py` 扫 `.tres` 抽出 441 条含中文 prose → `content.csv` (源串当 key, 列 keys,en), 全部已填 en; `build_translations.gd` 产出 `content.en.translation` 并在 project.godot 注册。显示处已包 `tr()`: 出身/事件(标题+正文+选项+历史)/融资轮名与解锁摘要/设施·供电·GPU spec 名/数据集模板名/产品类型 spec 名/科技节点(display_name·description·effects_summary, 经 research_dialog 与科技 tab; 科技 tab 的英文术语→中文替换表 `_sanitize_tech_text` 已改 zh-only)。守护测试 `tests/unit/content_translation_coverage_test.gd` 钉死每条 .tres 中文 prose 都有 en。
> **内容层 pass-through**: 玩家自命名 (模型/产品/自采数据集/公司名) 与化名 codename (NPC/arch) 不在 content.csv, `tr()` 原样返回; 少数低频 runtime 显示点 (owned dataset / NPC 榜行 / task 模板名) 未显式包 tr, en 下回落中文, 后续按需补 (译文已就绪)。
> **人名罗马化**: lead / 员工的中文真名 (`HiringSystem._gen_chinese_name` 随机拼出) 不进 content.csv (45 姓 × 60 名 = 组合爆炸, 且随机生成无法预译), 改由运行时 `NameRomanizer` 在非 zh locale 下转拼音 ("王伟"→"Wang Wei", "李婉清"→"Li Wanqing")。见 §12。

## 1. 设计目标

- 用 Godot 4 内置 `TranslationServer` + CSV 导入流程, 不自造 i18n 框架。
- **新写的人面字符串一律 `tr(...)`**, 不再硬编码中文; 旧代码逐 tab 迁移。
- 首次启动默认语言为英文 (`en`); 玩家可在设置里实时切到中文 (`zh_CN`) 或英文, 选择会持久化 (见 §11)。
- 翻译缺失时的 fallback 仍固定简体中文 (`zh_CN`), 避免英文表漏项时直接露出语义 key。
- 两层翻译源 (见 §2bis):
  - `resources/i18n/strings.csv` — **UI 框架文案**, 语义 key (`NAV_/ACTION_/...`), 列 `keys, zh, en`。
  - `resources/i18n/content.csv` — **游戏内容文案** (事件卡 / 科技 / 数据集 / 出身 / 基建 / NPC / 产品 / 任务 / 融资 / 角色等 `.tres` 里的 prose), **源中文串当 key**, 列 `keys, en` (key 自身即 zh, 由工具从 `.tres` 抽取生成)。
- 玩家可在设置里实时切换语言, 选择持久化到 `user://preferences.cfg` (见 §11)。

### 1.1 非目标
- 不做 RTL / 复杂复数规则 (中文/英文都不需要 ICU 级别)。
- 不做插件式词条扩展 (玩家自定义包之类), CSV 走天下。
- 不翻译化名 (动物 / 植物族 codename) — 这些是世界观一部分, 一律保持原文。
- 不随 locale 改数字 / 货币格式 (本轮固定 `$X` / `12,345` 中式格式)。

## 2. Key 命名约定

格式: `MODULE_SUBJECT_DETAIL`, 全大写下划线。

| 前缀 | 用途 | 示例 |
|---|---|---|
| `APP_` | 全局 / 顶级 | `APP_TITLE` |
| `NAV_` | 侧栏导航项 | `NAV_OPERATIONS` / `NAV_RND` / `NAV_MARKET` / `NAV_MODELS` / `NAV_INFRA` |
| `TOPBAR_` | 顶栏指标 | `TOPBAR_TURN` / `TOPBAR_CASH` / `TOPBAR_NET_CASHFLOW` / `TOPBAR_RANK` |
| `ACTION_` | 通用动作按钮 | `ACTION_NEW` / `ACTION_CANCEL` / `ACTION_CONFIRM` / `ACTION_DELETE` |
| `CARD_<DOMAIN>_` | 各类卡片字段与按钮 | `CARD_MODEL_PUBLISH` / `CARD_DC_BUILD_GPU` / `CARD_LEAD_HIRE` |
| `DRAWER_<DOMAIN>_` | 抽屉表单 | `DRAWER_PRETRAIN_TITLE` / `DRAWER_PRETRAIN_FIELD_ARCH` |
| `EMPTY_<DOMAIN>_` | 空状态 | `EMPTY_MODELS_TITLE` / `EMPTY_MODELS_HINT` |
| `STATUS_` | 模型 / 任务 / 数据中心状态徽章 | `STATUS_PRETRAINED` / `STATUS_EVALUATED` / `STATUS_PUBLISHED` |
| `TOAST_` | 提示 toast | `TOAST_SAVE_OK` / `TOAST_INSUFFICIENT_FUND` |

key 一旦发布**不可重命名** (会让翻译丢失); 改文案就改 CSV 的 value 列。废弃 key 在 CSV 末尾留注释 `# deprecated 2026-xx-xx <key>`, 实际行先保留一周再删, 防止热分支冲突。

---

## 2bis. 两层 key 策略

文案分两类, 用两套 key / 两份 CSV, 互不干扰:

| 层 | 适用 | key 形态 | 翻译源 | 调用 |
|---|---|---|---|---|
| **UI 框架** | 导航 / 按钮 / 对话框字段 / 空状态 / 状态徽章 / toast 等界面骨架 | 语义 key `NAV_/ACTION_/DRAWER_/EMPTY_/STATUS_/TOAST_…` (§2) | `strings.csv` (`keys, zh, en`) | `tr("NAV_OPERATIONS")` |
| **游戏内容** | `.tres` 资源里的 prose: 事件卡 title/body/option、科技 display_name/description、数据集 / 出身 / 基建 / NPC / 产品 / 任务 / 融资 / 角色的 display_name 等 | **源中文串本身当 key** | `content.csv` (`keys, en`; key 即 zh) | `tr(card.title)` (直接拿 `.tres` 里的中文串去查) |

**为什么内容层用源串当 key**:
- `.tres` 共 ~180 份, 源串当 key 意味着 **`.tres` 一行都不用改** — 显示处把现有读取包一层 `tr(...)` 即可。
- zh 永远正确: 查不到就返回 key 自身 (= 中文), 所以 zh locale 下不依赖 `content.csv` 完整。
- `content.csv` 只需 `en` 一列; 由 `tools/extract_content_strings.*` 从 `.tres` 扫描抽取所有 prose 串生成 `keys` 列, 人工 (或我) 填 `en`。
- 代价: 改了 `.tres` 里的中文原文会让对应 en 失配 → 该串回落显示中文。守护测试 (§7) 会扫出"有中文 prose 但 `content.csv` 无 en"的串, 列成未翻清单, 不让它悄悄漏掉。

**仍然不翻**: Log、`StringName` id / 枚举、化名 codename (`sparrow_7b` / `cypress_t3` / `ant_v3`)、设计文档 / 注释 / commit。

## 3. 翻译源

`resources/i18n/strings.csv` (UTF-8, 含 BOM 与否都行, Godot 都吃):

```csv
keys,zh,en
APP_TITLE,Scaling Up,Scaling Up
NAV_OPERATIONS,运营,Operations
NAV_RND,研发,R&D
NAV_MARKET,市场,Market
ACTION_NEW,新建,New
ACTION_CANCEL,取消,Cancel
```

- **逗号、引号**: 标准 CSV 转义 (`"含,逗号"`、双引号转义成 `""`)。
- **换行**: value 里写字面 `\n` 表示换行 (build_translations 会转成真换行); 不要在 CSV cell 里塞真实换行。
- en 列允许为空, 表示"还没翻"; `tr()` 在 en locale 下会回落 (见 §5)。
- **不在 CSV 里塞插值字段**: 用 `%s` / `%d` / `String.format` 在调用点处理 — 一来 Godot CSV pipeline 不支持 ICU MessageFormat, 二来调用点拼装也更易读。

例:
```gdscript
$Label.text = tr("CARD_MODEL_PARAM_COUNT") % [param_b]   # CSV: "参数量: %sB"
```

## 4. 注册与默认 locale

`project.godot` 的 `[internationalization]` 段:

```ini
[internationalization]
locale/translations=PackedStringArray("res://resources/i18n/strings.zh.translation", "res://resources/i18n/strings.en.translation")
locale/fallback="zh_CN"
```

Godot 导入 `strings.csv` 时按列名为每个 locale 生成一份 `strings.<locale>.translation`; 路径基于 `keys` 列的目录。

默认 locale = `en` (由 `Preferences.DEFAULT_LOCALE` 在首次启动时显式应用, 不依赖系统语言)。fallback locale 仍是 `zh_CN`, 用于英文译文缺失时回落到中文。运行时切语言用 `Preferences.set_locale(...)`。

---

## 5. 回落规则

Godot `tr()` 查找顺序:
1. 当前 locale 翻译表
2. fallback locale 翻译表 (这里固定 `zh_CN`)
3. 返回 key 本身

en 列空字符串视同未翻译, 自动走 step 2 回落到 zh。验收上: en locale 下 `tr("ACTION_CONFIRM")` 应返回中文 "确认" (而不是返回 key `ACTION_CONFIRM`), 因为 fallback 是 zh。**这条由单测固化, 不能改成"返回 key"**。

## 6. 何时该 `tr()`, 何时不该

**该走 tr()**:
- Label / Button / Tooltip 等出现在玩家屏幕上的字符串。
- 抽屉表单的字段名、按钮文案。
- 空状态文案。
- toast / 状态条消息。
- 错误提示 (能被玩家看见的那部分)。

### 6bis. 事件选项的"标签 + 后果预览"拼装

事件选项按钮文案由两段拼成: `tr(opt.label) + " — " + 后果预览`。两段分属不同翻译层, **必须各自先翻译再拼**, 不能拼好再 `tr()` (拼出来的复合串不是任何 key, 会整条回落中文):

- **标签** (`opt.label`) 是 `.tres` 里的中文 prose → 内容层, 由 `event_view` 显示处 `tr(...)` 走 `content.csv`。
- **后果预览** (`EventSystem.describe_option_consequence`) 是运行时按 effect 拼的 UI 文案 (含金额/订阅数插值) → UI 层, 走 `strings.csv` 的 `EVENT_CONSEQ_*` key (`tr("EVENT_CONSEQ_SPEND") % 金额`)。
- 接线: `main.gd._build_event_view_data` 把**裸** `opt.label` 与已翻译的 `consequence` 分别放进 dict; `event_view` 翻译裸 label 后再追加 consequence。`locale_changed` 触发 `main._refresh()` 整段重建, 两段都按新 locale 重算。

### 6ter. 代码里定义的人面文案 (不是 .tres prose) 走 strings.csv 语义 key

有一类文案不在 `.tres` 里、而是**写死在 `scripts/` 代码**中, 又会进 UI —— 比如账本收支分类 (`economy_system.REASON_CATEGORY` / `_categorize`)、预训练修正项明细 (`task_system._modifier_entry` 的标签)、租用云数据中心名 (`infra_system` 的 `云算力 ×N 卡`)、中立出身名 (`founder_system` 的 `无出身`)。

这类串**不能**塞进 `content.csv`: `tools/extract_content_strings.py` 只扫 `.tres`, 会把任何不在 `.tres` 里的 key 当孤儿删掉 (见脚本末尾 `dropped`)。所以它们一律走 **`strings.csv` 语义 key** (UI 框架层):

- 账本分类: `ECO_CAT_*` (值同时是 `weekly_ledger` 的分组 key, 改为语义 key 后显示处 `tr(key)`)。
- 修正项: `TASK_MOD_*` (`task_system` 存 key, `pretrain_dialog._format_modifier_lines` 显示时 `tr(label)`)。
- 云 DC 名: `INFRA_CLOUD_DC` (带卡数插值 `% count`, 用 `Datacenter.display_label()` 在显示时按 locale 拼, 不把本地化结果烤进存档)。
- 中立出身: `FOUNDER_NEUTRAL`。

**守护测试** `tests/unit/code_string_i18n_coverage_test.gd`: 扫 `scripts/` + `scenes/` 里**写死的中文串字面量**, 每条 (除 allowlist: Log / 注释 / `StringName` id / 化名 / 默认名 `创始人` / 语言按钮 `中文` / 截图夹具 / locale 守护过的 `_sanitize_tech_text` 替换表) 都必须可翻译 (= 命中 `strings.csv` 的 zh 列或 `content.csv` 的 key)。防止"代码里硬编码中文进 UI"这类漏翻 (正是事件后果预览 / 账本分类 / DC 名当初漏掉的盲区) 复发。

**不该走 tr()** (保持原中文 / 英文):
- Log 输出 (`Log.info/warn/error`) — 日志面向开发者, 不翻。
- 内部 `StringName` 标识符 (`&"chatbot"` 之类) — 这是 id 不是文案。
- 化名 (动物 / 植物族, 如 `sparrow_7b` / `cypress_t3`) — 世界观, 不翻。
- Resource 的 `display_name` 字段 (e.g. `FacilitySpec.display_name`): 暂时保持中文硬编码, 当对应 Resource 类下次大改时一起 i18n 化; 不在本轮触碰。
- 设计文档、注释、commit 信息 — 永远中文。

## 7. 测试约束

`tests/unit/localization_test.gd` 至少覆盖:

- `project.godot` 注册的翻译路径数量 ≥ 1, 且包含 `strings.zh.translation`。
- `strings.csv` 每条有效记录的列数必须等于表头列数; 含英文逗号的 value 必须用标准 CSV 引号包住, 防止 Godot 导入时把一条 key 拆成多于 2 个 locale。
- `tr("APP_TITLE")` 在 `zh_CN` locale 返回 "Scaling Up"。
- `tr("NAV_OPERATIONS")` 在 `zh_CN` 返回 "运营", 在 `en` 返回 "Operations"。
- `tr("__no_such_key_dont_define__")` 返回 key 本身。
- en 列空时回落 zh: 加一条 seed key `_PIPELINE_FALLBACK_SMOKE` 仅填 zh, 验证 en locale 下仍返回中文。
- locale 切换可逆: `set_locale("en") → set_locale("zh_CN")` 后 `tr("NAV_OPERATIONS")` 回到 "运营"。

`tests/unit/preferences_test.gd` (对应 §11) 覆盖:

- `Preferences.set_locale("en")` 后 `TranslationServer.get_locale()` == `"en"`, 且 `EventBus.locale_changed` 以 `"en"` emit。
- 写盘后重新 `_load()` 能读回所选 locale (持久化往返)。
- locale 未变时 `set_locale` 不重复 emit。

`content.csv` 守护 (对应 §2bis, 阶段 3): 扫描 `.tres` 的 prose 字段, 断言每条中文串在 `content.csv` 有非空 en (或落在显式 allowlist), 否则报未翻清单。

后续每加一个组件 / view, 集成测试同时验证 *人面字符串走 tr*。

## 8. 工作流（怎么加 / 改翻译）

两条轨道。改完都要跑一次生成命令 + 测试。

**生成命令**（把 CSV 编译成游戏读的 `.translation`，产物提交进库）:
```
godot --headless --path . -s tools/build_translations.gd
```

### A. UI 文案（按钮 / 标题 / 字段 / toast 等）

1. 在 `resources/i18n/strings.csv` 加一行 `KEY,中文,English`（语义 key，全大写下划线，见 §2）。含换行写字面 `\n`。
2. 代码里 `tr("KEY")`；带变量用 `tr("KEY") % [...]`（`%s`/`%d` 在调用点拼）。
3. `const` 数组 / 字典里不能调 `tr()`（没 `self`）——存 key、取用处再 `tr()`；纯 `static` 脚本用 `TranslationServer.translate("KEY")`。
4. 跑生成命令 + 测试。断言文案的测试断 `tr("KEY")`，别写死中文。

### B. 游戏内容文案（`.tres` 里的事件 / 科技 / 出身 / 设施等 prose）

1. 直接在 `.tres` 里写中文（照常）——**不手动碰 content.csv**。
2. 抽取：`python3 tools/extract_content_strings.py`（扫所有 `.tres` 把含中文的串同步进 `content.csv`，保留已填的 en）。
3. 在 `content.csv` 填 `en` 列。**别手写引号/逗号转义**（Godot 解析器比 Python 宽松会出错）——要批量改就用 Python `csv` 模块，或只动 en 单元格。
4. 显示处把读取包一层 `tr(...)`，例如 `tr(card.title)`、`tr(spec.display_name)`。`tr()` 命中 key 才翻、否则原样返回，所以包玩家自命名 / 化名也安全。
5. 跑生成命令；`content_translation_coverage_test.gd` 会保证每条中文 prose 都有 en。

### 测试
```
godot --headless --path . -s addons/gut/gut_cmdln.gd -gexit
```
测试基线 locale 钉在 `zh_CN`（`tests/support/gut_cjk_font_hook.gd`），所以断言中文文案的旧测试照常通过；切 locale 的测试自己恢复 zh_CN。

## 11. 运行时语言切换与持久化

玩家可在**设置**里实时切换中文 / 英文, 选择跨启动持久化。设置入口有两个、共用同一个对话框:

- **起始页 → 设置** (开局前);
- **游戏内顶栏 → 设置** (`main.gd` 顶栏「存档」左侧的「设置」按钮, 让玩家不必退回起始页就能改语言)。

### 11.0 SettingsDialog 组件

`scenes/ui/settings_dialog/settings_dialog.gd` (`extends AcceptDialog`) — 设置对话框唯一实现, 起始页与游戏内顶栏都 `new()` 它再 `open()`。内容不再是裸控件纵向堆叠, 而是三块分区面板:

- **显示** (`SETTINGS_SECTION_DISPLAY`): 语言、全屏、界面缩放。
- **音频** (`SETTINGS_SECTION_AUDIO`): 背景音乐、音乐音量、界面音效。
- **系统** (`SETTINGS_SECTION_SYSTEM`): 自动存档、返回主菜单入口。

每个分区用 `PanelContainer` 承载, 白底 / 浅灰边 / `R_MD` 圆角 / `S_4` 内边距; 分区标题使用 bold 字体和小型抽象图标, 控件保持两列或紧凑行布局, 让玩家能快速扫读。功能项包括:

- **语言**: 「中文 / English」两按钮做成分段控件, 当前语言对应按钮置灰; 点击走 `Preferences.set_locale(...)`。
- **全屏**: `CheckButton` (`SETTINGS_FULLSCREEN`), 初值读 `Preferences.fullscreen`, 切换走 `UITheme.set_fullscreen(on)` (持久化 + `DisplayServer` 切窗口模式)。
- **界面缩放**: `OptionButton` (`SETTINGS_UI_SCALE`), 选项「自动 / 100% / 125% / 150% / 175% / 200%」, 初值读 `Preferences.ui_scale` (`0`=自动→`SETTINGS_UI_SCALE_AUTO`)。切换走 `UITheme.set_ui_scale(v)` (持久化 + 重应用 `content_scale_factor`)。详见 `UI视觉系统设计.md §9bis`。
- **背景音乐**: `CheckButton` (`SETTINGS_MUSIC`) + 「音乐音量」`HSlider` (`SETTINGS_MUSIC_VOLUME`, 0..1)。开关走 `MusicPlayer.set_enabled`, 滑块走 `MusicPlayer.set_volume` (单消费者, 不走 EventBus, 见 `音频系统设计.md §4`)。
- **界面音效**: `CheckButton` (`SETTINGS_SFX`), 初值读 `SfxPlayer.is_enabled`, 切换走 `SfxPlayer.set_enabled` (按钮点击 SFX 与按压反馈见 `音频系统设计.md §4bis`)。
- **自动存档**: `CheckButton`, 初值读 `TurnManager.autosave_enabled`, 切换即写回。
- **返回主菜单** (仅游戏内): 一个 `bool allow_return_to_menu` 开关控制其可见性 —— 起始页打开时为 `false` (已在菜单, 不显示), 游戏内顶栏打开时由 `main.gd` 置 `true` 后再 `open()`。点击先弹 `ConfirmationDialog` (文案: 进度将自动存档, 可从「继续游戏」恢复), 确认后 `emit return_to_menu_requested` 并关闭自身。对话框**不**直接碰 `Save` / 切场景 —— 真正的「存档 + 切回起始页」由订阅信号的 `main.gd` 执行 (见 `出身系统设计.md §1`), 保持组件与导航/存档解耦。
- 自身订阅 `EventBus.locale_changed` → 刷新 title / 标签 / 关闭按钮 / 语言按钮置灰态 / 返回主菜单按钮文案, 所以在对话框开着时切语言, 对话框文案也即时变。

### 11.1 Preferences 单例

`scripts/autoload/preferences.gd` (autoload 名 `Preferences`) — 用户偏好持久化, **独立于游戏存档** (`Save` 管游戏状态, `Preferences` 管跨存档的偏好):

- `var locale: String` — 当前语言, 默认 `en`；玩家切换后写入 `[i18n] locale`, 以后启动优先读保存值。
- `var music_enabled: bool` / `var music_volume: float` / `var sfx_enabled: bool` — 背景音乐开关 + 线性音量 (默认见 `Preferences` 常量) + 界面音效开关, 落 `[audio]` 段。
- `var fullscreen: bool` / `var ui_scale: float` — 全屏开关 + 界面缩放档 (`0.0`=自动), 落 `[display]` 段。
- `set_locale(loc)` — 设 `TranslationServer.set_locale(loc)` + 写盘 + `EventBus.locale_changed.emit(loc)`。语言一致时早退, 不重复刷。
- `set_music_enabled / set_music_volume / set_sfx_enabled / set_fullscreen / set_ui_scale` — 值未变早退; 变了仅持久化 (单消费者, 不广播)。
- `_ready()` 启动时 `_load()` 读回上次选择并 `_apply_locale()`; 测试运行 (`-gtest`/`gut_cmdln`) 下不自动套用, 保持测试 hermetic。
- 落盘: `user://preferences.cfg` (`ConfigFile`, section `i18n`/`audio`/`display`)。`_save()` 先 `load` 再 `set_value`, 不覆盖其它偏好。

### 11.2 locale_changed 信号与实时刷新

- `EventBus.locale_changed(locale: String)` — 过去式广播, emit-and-forget (同 `系统耦合矩阵.md §4`)。
- `main.gd` 订阅 `locale_changed` → 重跑 `_refresh()` (重建 14 个 tab) + 顶栏 + 侧栏导航文案。因为 view 都是 "pull" 模型、`refresh()` 时整段重建, 重渲染即拿到新 locale 下的 `tr(...)`。
- `SettingsDialog` 的「中文 / English」按钮调 `Preferences.set_locale(...)` 而非裸 `TranslationServer`, 这样既持久化又触发刷新; 对话框自身也订阅 `locale_changed` 实时刷新文案。

### 11.3 启动 locale 决策

启动顺序: `Preferences._ready()` 读盘 → 有保存值就用; 无保存值走 `Preferences.DEFAULT_LOCALE` (`en`) 并 `_apply_locale()`。不依赖系统语言自动判定。`project.godot` 的 `locale/fallback` 继续保持 `zh_CN`, 只负责译文缺失时的显示回落。

## 12. 人名罗马化 (拼音)

lead / 员工的 `display_name` 是 `HiringSystem._gen_chinese_name()` 从「姓池 × 名池」随机拼出的中文真名 (如 "王伟" / "李婉清"), **不是**化名 codename。它们既不该当 codename 原样保留 (英文 locale 下满屏中文出戏), 也塞不进 `content.csv` (45 姓 × 60 名 ≈ 组合爆炸, 而且运行时随机生成, 无法预先翻译)。

方案: 运行时按 locale 转**拼音**, 不改存档 (`display_name` 永远存中文, 显示时转), 老档与切换 locale 都自然生效。

### 12.1 NameRomanizer

`scripts/systems/name_romanizer.gd` (`class_name NameRomanizer`, 纯静态工具):

- `static func romanize(name) -> String` — 中文真名 → 首字母大写拼音。首字 = 姓, 其余 = 名; 姓与名各自首字母大写, 名内多字相连 ("李婉清" → "Li Wanqing", "张子墨" → "Zhang Zimo")。
- `static func localized(name) -> String` — locale 感知: `TranslationServer.get_locale()` 以 `zh` 开头时原样返回中文, 否则 `romanize`。**显示处一律调它**, 不直接用 `lead.display_name`。
- 内部 `_PINYIN` 字典只覆盖姓 / 名池里出现的字 (120 个)。**遇到表外字 → 整串原样返回** (passthrough): 这样玩家自取的创始人名 / 默认 "创始人" / 英文名都不会被半翻坏。

### 12.2 显示处接线

所有露出 lead 名的地方包一层 `NameRomanizer.localized(...)`:
`lead_card.gd` (title + avatar fallback)、`main.gd._marketing_lead_label`、`new_campaign_dialog` / `posttrain_dialog` / `new_product_dialog` / `dataset_collection_dialog` 的 lead 下拉。locale 切换走 `main.gd` 的 `_refresh()` 整段重建, 重渲染即拿到新 locale 拼音。

### 12.3 守护测试

`tests/unit/name_romanizer_test.gd` 钉死: (1) 代表性中文名 → 期望拼音; (2) zh locale 下 `localized` 返回中文、en 下返回拼音 (测后恢复 `zh_CN`); (3) **姓池 / 名池里每个字都在 `_PINYIN` 有非空映射** — 防止以后往池里加字却忘了补拼音, 导致该名在 en 下 passthrough 露中文。
