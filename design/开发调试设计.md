# 开发调试设计

> **目的**: 定义 *Scaling Up* 的日志基础设施 — 一个统一、可过滤、自动带类别的日志系统, 开发期辅助排错, 测试期可静默。
> **读者**: 任何在 system / UI 代码里写日志的人; 想排查跨系统问题或断言"应当产生过 X 类别的 ERROR"的人。
> **状态**: 🟡 v0 已落地 (§2-§4 + §9-§10), §6/§7/§8 列出**未实现**的后续方向。

---

## 1. 目标

| 目标 | 含义 | 验收 |
|---|---|---|
| **多维过滤** | 全局开关 + 级别阈值 + 类别白/黑名单 | 关掉 `tasks` 类别后, TaskSystem 的日志完全消失 |
| **零 `print()`** | 项目里禁止裸 `print` / `printerr`, 统一走 `Log.*` | `grep -rn 'print(' scripts/ | grep -v '_test.gd'` 为空 |
| **测试静默** | 测试运行时 stdout 不被日志淹没, 但日志可被断言 | 单测能读到 `Log.captured`, stdout 干净 |
| **失败路径安静** | 预期内的错误分支先做存在性 / 格式校验, 再返回 `{ok=false, error=...}` | unknown `.tres` / 损坏 JSON 测试不产生引擎级红字 |
| **字体启动安静** | UI 字体在 autoload 阶段装入默认主题, 再创建中文控件; GUT runner 输出也改用同一 CJK 字体 | debug play / UI 测试 / 全量 GUT 不因默认 Open Sans 或插件字体缺 CJK 而刷 FreeType 红字 |
| **跨系统可观测** | 看到一次玩家操作触发的所有 system 反应 | 开 `info` 级别即可看到 economy → tasks → research 的串联 |

## 2. `Log` Autoload

注册顺序:

```
GameState → EventBus → TurnManager → Log → UITheme → CommandBus → (业务 system)
```

- `Log` 自身不依赖 EventBus, 但排在它后面, 让后续 `BusInspector` (§7, 未实现) 可挂订阅。
- `UITheme` 排在 `Log` 后, 负责把 `assets/fonts/cjk.ttf` 装进 `ThemeDB.get_default_theme()`, 让"先设文本、后 add_child"的调试 UI 不会先走 Godot 内置 Open Sans 而刷红字。
- `UITheme.apply_font_to_theme()` 必须同时覆盖普通控件的 `font` 与弹窗/窗口的 `title_font` (`Window` / `AcceptDialog` / `ConfirmationDialog` / `FileDialog` / `PopupPanel`)。否则中文弹窗标题会回退到 Godot 默认字体, macOS/headless 下 system fallback 可能得到空字体路径并刷 `FreeType: Error loading font: ''`。
- 运行时再次显式关闭 UI 字体的 system fallback (`allow_system_fallback=false`)。缺字宁可显示 tofu 或由后续字体替换解决, 不让 Godot 在测试与 debug play 中查系统字体。
- GUT 自带的 CourierPrime / AnonymousPro 字体不覆盖中文。`.gutconfig.json` 必须注册 `tests/support/gut_cjk_font_hook.gd` 作为 `pre_run_script`, 在测试开始前把 runner 的 `RichTextLabel` 输出槽位 (`font` / `normal_font` / `bold_font` / `italics_font` / `bold_italics_font`) 以及编辑器面板使用的 `TextEdit.font` 全部换成 `UITheme` CJK 字体, 避免中文 pending/assert 文案触发插件字体的 system fallback。

### 2.1 公共 API

`Log` 暴露五个级别方法 `trace/debug/info/warn/error(category, message, data)` + 过滤 / 捕获相关字段 (`enabled` / `min_level` / `category_filter` / `captured` / `console_enabled`)。**完整声明以 `log.gd` 为准**, 本文不重复签名。

`data` 参数是**强约定的** — 不要把数据拼到 `message` 字符串里, 让格式化器决定怎么呈现 (console 一行 JSON, 未来 sink 可解析)。

### 2.2 调用方约束

```gdscript
# 推荐 ✓
Log.debug(&"economy", "paid salaries", {&"total": 1500, &"count": 5})

# 反模式 ✗ (不要把数据拼字符串)
Log.debug(&"economy", "paid salaries: total=%d count=%d" % [1500, 5])
```

## 3. 标准输出格式

```
[LEVEL] [category] message {data}
```

例:

```
[INFO ] [economy ] paid salaries {"total":1500,"count":5}
[DEBUG] [economy ] applied weekly burn {"rent":3000,"compute":1200}
```

类别字段固定宽 8, 方便肉眼对齐。**当前 v0 不抓 caller 文件名/行号** (`get_stack()` 性能成本未评估), 排查靠 `category + message` 关键字定位。

## 4. 级别与类别过滤

### 4.1 级别

```
TRACE   极细粒度 (每个 bus 信号、每个 RNG 调用)
DEBUG   开发期默认级别 (系统内部状态变化)
INFO    重要事件 (回合开始/结束、玩家关键操作、存档完成)
WARN    可恢复异常 (无主请求被忽略、配置缺字段走默认)
ERROR   错误 (请求失败被 UI 看到、存档损坏、断言失败)
SILENT  关闭所有
```

### 4.2 类别 (基于实际代码使用)

| 类别 | 主要发声方 |
|---|---|
| `system` | autoload `_ready` / `_exit_tree` |
| `turn` | TurnManager (回合相位推进) |
| `economy` | EconomySystem (现金、薪资、融资、贷款、利息) |
| `hiring` | HiringSystem (招聘、签约、解雇) |
| `research` | ResearchSystem (模型生命周期) |
| `tasks` | TaskSystem (任务启动 / 推进 / 完成) |
| `tech` | TechTreeSystem (科技节点解锁) |
| `infra` | InfraSystem (DC / GPU / 部署) |
| `dataset` | DatasetSystem |
| `market` | MarketSystem (排行榜 / NPC) |
| `user` | UserSystem (订阅 / 流失) |
| `product` | ProductSystem |
| `monetization` | MonetizationSystem (订阅与 API 收入) |
| `marketing` | MarketingSystem (Campaign) |
| `event` | EventSystem |
| `save` | SaveSystem (存档 / 读档) |
| `bus` | (未实现, 见 §7) EventBus 流量自动追踪 |
| `ui` | 各 UI 场景 |
| `test` | 测试桩自身 |

新增 system 时**必须**在本表登记类别。类别名永远 `StringName` 字面量。

## 5. Sinks (当前形态)

v0 内置 2 个 sink, 不可扩展:

- **`captured`**: 永远开。`Log.captured: Array[Dictionary]` 让测试断言 "应当产生过 X 类别的 WARN+"。
- **`console`**: `console_enabled = true` 时 `print(_format(record))`; 测试 runner (cmdline 含 `-gtest` / `--test` / `gut_cmdln.gd`) 启动时自动改为 `false` + `min_level = WARN`。

未来若需要 FileSink / OverlaySink / 远程 sink, 把单 sink 改成 `Array[LogSink]` 即可, 接口契约见 §12。

## 6. 运行时配置

当前**没有**配置文件 / 命令行参数 / 快捷键支持。调级别 / 关类别要在测试里直接改 `Log.min_level` / `Log.category_filter`。详细的多源配置 (`debug_default.cfg`, `user://debug_config.json`, `--log-level`, F2/F3/F4) 留作未来扩展, 见 §12。

## 7. 与 EventBus 集成 (未实现)

`BusInspector` 在 `category_filter[&"bus"] == true` 时自动 trace 所有 EventBus 信号, 是排查跨系统问题的关键能力。当前 `bus` 类别空 stub, 待实现, 详见 §12。

> 注: 项目用 `CommandBus.send/register` 模式承载玩家命令 (见 [命令总线表.md](命令总线表.md)), 与原"BusRequest mark_handled" 设想不同; BusInspector 未来需同时挂 EventBus 信号与 CommandBus 调用。

## 9. 编码规范

| 规则 | 备注 |
|---|---|
| 严禁 `print` / `printerr` / `print_debug` / `print_rich` | 测试代码也不允许; 用 `Log.*` |
| 严禁 `push_error` / `push_warning` | 业务错误用 `Log.error` |
| 预期失败路径先做显式校验 | 缺失 `.tres` 用 `ResourceLoader.exists` 拦截; 损坏 JSON 用 `JSON.parse` 返回码拦截 |
| UI helper 标注具体返回类型 | 返回 Product / Model / Resource 的函数不要让调用方拿 `Variant` 再访问字段 |
| GDScript warning 视为要清理的噪音 | 不用和 `class_name` 同名的 preload 常量, 直接使用全局类名; 自定义信号用 `signal_name.emit(...)` 显式发射; `Node` 派生脚本参数不要命名为 `name`; 整数除法显式转 `float` 或用 `floori`/`roundi` 表达取整意图; 未使用变量删除; 未使用参数加 `_` 前缀; 对外信号要么 emit 要么用 `@warning_ignore` 说明 |
| 类别用 `&"..."` 字面量 | 不允许 `String`, 不允许动态拼接 |
| 数据进 `data` 参数 | 不要塞 `message` 字符串 |
| `error` 调用前应当先发出 `WARN` (预警) 或直接 `ERROR` (已失败) | 没有"info 级的错误" |

CI / pre-commit 检查 (未来):

```
grep -rn 'print(' scripts/ | grep -v '_test.gd'    # 必须为空
grep -rn 'push_error' scripts/                      # 必须为空
```

## 10. 测试中的行为

测试 runner 启动时 `Log._ready` 自动:
- `console_enabled = false` (stdout 不被淹没)
- `min_level = WARN` (只记 WARN+)

`captured` 永远开, 测试可对 `Log.captured` 做断言 (按 `level` / `category` 过滤, 验证"应当产生过 X 类别的某级别记录")。实际写法见 `tests/unit/` 下用到 `Log.captured` 的测试。测试若要清空 `captured`, 在 `before_each` / `after_each` 自己处理。

---

## 11. 性能与稳定性

- 早退路径 (未通过 enabled / level / category 过滤) 是 3 次字典 / 比较, 单调用 < 0.1µs。
- 当前 v0 不抓 caller frame, 性能开销最小。未来若加 `get_stack()`, **必须先过滤再抓栈**。
- sink 内部失败不能反过来打断业务流程, sink 内部 `try` / 静默失败。
